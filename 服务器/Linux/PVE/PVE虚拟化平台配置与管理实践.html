# PVE虚拟化平台配置与管理实践

Proxmox VE(PVE)是一款基于Debian的开源虚拟化平台，支持KVM和LXC容器。本文整理了在使用PVE过程中遇到的一些配置问题和解决方案。

# 存储管理

# 合并local-lvm到local

PVE默认将磁盘分为local和local-lvm两个存储，可以将它们合并以获得更灵活的空间管理。

操作步骤：

  1. 备份所有虚拟机
  2. 删除所有虚拟机
  3. 删除local-lvm逻辑卷：
    lvremove pve/data
  4. 将空间分配给local：
    lvextend -l +100%FREE -r pve/root
  5. 在Web界面删除local-lvm存储： 数据中心 → 存储 → 删除local-lvm
  6. 编辑local存储，在内容中添加"磁盘映像"和"容器"
  7. 恢复虚拟机

注意： 此操作会丢失local-lvm中的所有数据，务必先备份。

# 虚拟机磁盘扩容

为虚拟机增加磁盘空间的完整流程。

1. 查看磁盘信息

# 查看磁盘和分区
lsblk
fdisk -l

2. 创建swap分区

fdisk /dev/sda

# 在fdisk交互界面中：
# p - 查看当前分区
# n - 创建新分区
# 选择主分区类型 (p)
# 输入分区大小，如 +4g
# t - 转换分区类型
# 82 - 设置为swap类型
# p - 确认分区创建成功
# w - 保存并退出

3. 格式化并挂载swap

# 更新磁盘信息
partprobe /dev/sda

# 格式化为swap
mkswap /dev/sda2

# 查看UUID
blkid

# 编辑fstab自动挂载
vim /etc/fstab
# 添加一行：UUID=xxx none swap sw 0 0

# 激活swap
swapon -a

# 验证
free -h

# 虚拟机管理

# 虚拟机备份和迁移

PVE提供了便捷的虚拟机备份功能。

Web界面操作：

  1. 选择虚拟机 → 备份 → 立即备份
  2. 备份文件默认存储在 /var/lib/vz/dump/
  3. 恢复时选择备份文件即可

导出虚拟机配置：

# 虚拟机配置文件位置
/etc/pve/qemu-server/<VMID>.conf

# 复制到其他PVE主机
scp /etc/pve/qemu-server/100.conf root@target-host:/etc/pve/qemu-server/

# 安装群晖NAS虚拟机

PVE上可以运行黑群晖系统，实现软NAS功能。

基本步骤：

  1. 下载群晖引导文件和DSM系统镜像
  2. 创建虚拟机，配置合适的CPU和内存
  3. 导入引导盘镜像
  4. 添加数据硬盘（可以直通物理硬盘）
  5. 启动虚拟机，使用Synology Assistant查找并安装

硬盘直通示例：

# 查看硬盘by-id
ls -l /dev/disk/by-id/

# 添加硬盘到虚拟机
qm set <VMID> -sata1 /dev/disk/by-id/ata-xxx

参考文档：PVE安装群晖教程 (opens new window)

# 网络配置

# WiFi连接配置

PVE默认不支持WiFi，需要安装connman来管理无线网络。

安装connman：

apt update
apt install connman

配置文件位置：

/var/lib/connman/

# 软件源配置

# 移除企业订阅源

PVE默认启用企业源，没有订阅会报错。可以切换到社区源。

# 删除企业源
rm -rf /etc/apt/sources.list.d/pve-enterprise.list

# 下载密钥
wget http://download.proxmox.com/debian/proxmox-ve-release-6.x.gpg \
  -O /etc/apt/trusted.gpg.d/proxmox-ve-release-6.x.gpg

# 添加社区源（PVE 6）
echo "deb http://download.proxmox.com/debian/pve buster pve-no-subscription" \
  > /etc/apt/sources.list.d/pve-install-repo.list

# 更新
apt update

# 配置国内镜像源

编辑 /etc/apt/sources.list，替换为国内镜像：

# 清华源
deb https://mirrors.tuna.tsinghua.edu.cn/debian/ buster main contrib non-free
deb https://mirrors.tuna.tsinghua.edu.cn/debian/ buster-updates main contrib non-free
deb https://mirrors.tuna.tsinghua.edu.cn/debian-security buster/updates main contrib non-free

# 常见问题

# CPU卡死问题

部分环境下虚拟机可能出现CPU占用异常的问题。

解决方法：

# 移除resolvconf
apt purge resolvconf

# 配置DNS
echo "nameserver 127.0.0.1" > /etc/resolv.conf

# 性能优化建议

# 虚拟机配置优化

  1. CPU设置：

    • 选择host CPU类型获得更好性能
    • 合理分配CPU核心数，避免过度分配

  2. 内存管理：

    • 启用ballooning动态调整内存
    • 为宿主机保留足够内存

  3. 磁盘性能：

    • 使用VirtIO驱动
    • 启用discard（SSD）
    • 考虑使用缓存策略

  4. 网络优化：

    • 使用VirtIO网卡
    • 开启多队列支持
