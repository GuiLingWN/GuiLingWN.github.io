# 青龙面板部署与任务管理实践

青龙面板是一个支持Python、JavaScript、Shell等脚本的定时任务管理平台。本文记录了使用Docker部署青龙面板和配置自动化任务的实践过程。

# Docker部署

# 拉取镜像

# 拉取最新版本
docker pull whyour/qinglong:latest

# 更新镜像
docker pull whyour/qinglong:latest

# 启动容器

普通服务器：

docker run -dit \
   -v $PWD/ql/config:/ql/config \
   -v $PWD/ql/log:/ql/log \
   -v $PWD/ql/db:/ql/db \
   -v $PWD/ql/scripts:/ql/scripts \
   -p 5700:5700 \
   --name qinglong \
   --hostname qinglong \
   --restart always \
   whyour/qinglong:latest

新版本（推荐）：

docker run -dit \
   -v $PWD/ql/config:/ql/data/config \
   -v $PWD/ql/log:/ql/data/log \
   -v $PWD/ql/db:/ql/data/db \
   --net host \
   --name qinglong \
   --hostname qinglong \
   --restart always \
   whyour/qinglong:latest

NAS群晖等设备：

docker run -dit \
   -v /volume2/docker/ql/data:/ql/data \
   --net host \
   --name qinglong \
   --hostname qinglong \
   --restart unless-stopped \
   whyour/qinglong:latest

# 初始配置

# 首次登录

  1. 访问：http://<服务器IP>:5700
  2. 使用默认账号登录：admin / adminadmin
  3. 系统提示已初始化密码

# 查看初始密码

# 查看配置文件中的密码
docker exec -it qinglong cat /ql/config/auth.json

# 或在挂载目录中查看
cat $PWD/ql/config/auth.json

# 基本命令

# 容器管理

# 更新青龙
docker exec -it qinglong ql update

# 重启青龙（更新并编译）
docker exec -it qinglong ql restart

# 删除容器
docker rm -f qinglong

# 任务管理

# 拉取脚本仓库
docker exec -it qinglong ql repo <repourl> <path> <blacklist> <dependence> <branch>

# 拉取单个脚本
docker exec -it qinglong ql raw <fileurl>

# 立即执行脚本
docker exec -it qinglong task test.js now

# 并行执行脚本
docker exec -it qinglong task test.js conc

# 导出助力码
docker exec -it qinglong ql code

# 测试通知
docker exec -it qinglong notify test test

# 日志管理

# 删除7天前的日志
docker exec -it qinglong ql rmlog 7

# 删除30天前的日志
docker exec -it qinglong ql rmlog 30

# 拉取脚本仓库

# 命令格式

ql repo <repourl> <path> <blacklist> <dependence> <branch>

参数说明：

  • repourl: 仓库地址
  • path: 拉取哪些脚本（匹配规则）
  • blacklist: 不拉取哪些（黑名单）
  • dependence: 依赖文件
  • branch: 分支名称（可选）

# 示例仓库

通用脚本库：

# 国外网络
ql repo https://github.com/KingRan/JDJB.git "jd_|jx_|jdCookie" "activity|backUp" "^jd[^_]|USER|utils|function|sendNotify|ql|JDJR"

# 国内网络（使用代理）
ql repo https://ghproxy.com/github.com/KingRan/JDJB.git "jd_|jx_|jdCookie" "activity|backUp" "^jd[^_]|USER|utils|function|sendNotify|ql|JDJR"

拉取单个脚本：

# 使用raw命令
ql raw https://raw.githubusercontent.com/author/repo/main/script.js

# 使用国内代理
ql raw https://ghproxy.com/https://raw.githubusercontent.com/author/repo/main/script.js

# Telegram Bot配置

青龙面板支持通过Telegram Bot进行交互式管理。

# 安装Bot

# 最新版本直接执行
docker exec -it qinglong ql bot

# 或在容器内执行
ql bot

# 配置Bot

创建config/bot.json文件：

{
    "user_id": 123456789,
    "bot_token": "your_bot_token",
    "api_id": "your_api_id",
    "api_hash": "your_api_hash",
    "proxy": false,
    "StartCMD": true,
    "noretry": false
}

# 获取配置参数

A. 申请Telegram Bot：

  • 找@BotFather创建机器人
  • 获取bot_token

B. 获取user_id：

  • 找@userinfobot获取你的user_id

C. 获取api_id和api_hash：

  • 访问https://my.telegram.org/apps
  • 创建应用获取

# 启用Bot

# 启动Bot
docker exec -it qinglong ql bot

# 配置开机自启
# 编辑config.sh
# 将AutostartBot=""改为AutostartBot="true"

# Bot快捷菜单

找@BotFather发送/setcommands，粘贴以下内容：

getcookie - 扫码获取cookie
bean - 获取收支
chart - 统计收支变化
cmd - 执行cmd命令
edit - 编辑文件
dl - 下载文件
getfile - 获取目录下文件
log - 选择日志
node - 执行js脚本文件
set - BOT设置
setshort - 设置自定义按钮
snode - 选择脚本后台运行
start - 开始使用本程序

参考教程：青龙Bot配置详解 (opens new window)

# 数据迁移

# 迁移到NAS

# 在NAS上启动新容器
docker run -dit \
   -v /volume2/docker/ql/data:/ql/data \
   --net host \
   --name qinglong \
   --restart unless-stopped \
   whyour/qinglong:latest

# 从旧服务器复制数据
scp -r root@old-server:/root/ql/data/ /volume2/docker/ql/

# 批量授权文件
chmod -R 777 /volume2/docker/ql/data

# 使用压缩包迁移

# 在旧服务器压缩
tar -czvf data.tar.gz /root/ql/data/

# 传输到新服务器
scp data.tar.gz root@new-server:/volume2/docker/ql/

# 解压
cd /volume2/docker/ql/
tar -xzvf data.tar.gz

# Cookie管理

# 添加Cookie

  1. 登录青龙面板
  2. 进入"环境变量"页面
  3. 添加变量：
    • 名称：JD_COOKIE
    • 值：从浏览器获取的cookie
  4. 保存并启用

# 多账号管理

每个账号一个cookie，使用相同的名称JD_COOKIE，系统会自动识别为多个账号。

# 定时任务

# 添加定时任务

  1. 进入"定时任务"页面
  2. 点击"添加任务"
  3. 填写：
    • 任务名称
    • 命令：task script.js
    • Cron表达式：0 0 * * *（每天0点）
  4. 保存并启用

# Cron表达式示例

# 每天凌晨2点
0 2 * * *

# 每6小时
0 */6 * * *

# 每天9点和21点
0 9,21 * * *

# 每隔5分钟
*/5 * * * *

# 安全建议

  1. 修改默认端口：

    • 映射端口时使用非标准端口

  2. 使用强密码：

    • 登录后立即修改密码

  3. 限制访问：

    • 使用防火墙限制访问IP
    • 不要将面板暴露到公网

  4. 定期备份：

    • 定期备份data目录
    • 包含配置、数据库和脚本

  5. 更新维护：

    • 定期更新青龙版本
    • 及时更新脚本仓库
