# WSL2配置与使用技巧

WSL2（Windows Subsystem for Linux 2）让Windows用户能够直接运行Linux环境。本文整理了WSL2的安装配置、磁盘迁移和网络访问等实用技巧。

# 安装WSL2

# 启用必要的Windows功能

方法一：使用GUI

  1. 按Win+R，输入optionalfeatures打开Windows功能
  2. 勾选以下选项：
    • 适用于Linux的Windows子系统
    • 虚拟机平台
  3. 重启计算机

方法二：使用PowerShell（管理员）

# 启用WSL
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# 启用虚拟机平台
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 重启系统

# 安装WSL2

# 设置WSL2为默认版本
wsl --set-default-version 2

# 列出可用的Linux发行版
wsl --list --online

# 安装Ubuntu（示例）
wsl --install -d Ubuntu-20.04

# 常见安装问题

错误代码：0x80070424

这个错误通常是因为Windows Update服务未运行。

解决方法：

  1. 下载最新的WSL2 Linux内核更新包
  2. 访问：https://github.com/microsoft/WSL/releases
  3. 安装后重试

参考：WSL Release页面 (opens new window)

# 磁盘迁移

WSL2默认安装在C盘，随着使用会占用大量空间。可以将其迁移到其他盘。

# 迁移步骤

# 1. 查看当前WSL发行版
wsl --list --verbose

# 2. 导出发行版到tar文件
wsl --export Ubuntu-20.04 D:/ENV/wsl/ubuntu20.04-backup.tar

# 3. 注销当前发行版
wsl --unregister Ubuntu-20.04

# 4. 创建目标目录
mkdir D:\ENV\wsl\Ubuntu-20.04 -Force

# 5. 导入到新位置
wsl --import Ubuntu-20.04 D:/ENV/wsl/Ubuntu-20.04 D:/ENV/wsl/ubuntu20.04-backup.tar

# 6. 设置默认用户（重要！）
ubuntu2004 config --default-user your_username

# 或设置为root
ubuntu2004 config --default-user root

# 7. 验证迁移成功
wsl -d Ubuntu-20.04

# 8. 删除备份文件（可选）
Remove-Item D:/ENV/wsl/ubuntu20.04-backup.tar

# 注意事项

  1. 默认用户重置： 导入后默认用户会变成root，需要手动设置回原用户
  2. 备份重要数据： 迁移前务必备份重要文件
  3. 磁盘空间： 确保目标磁盘有足够空间
  4. 路径格式： 使用正斜杠/或反斜杠\都可以

# 网络配置

# 内网访问WSL2

WSL2使用虚拟网络，内网设备默认无法直接访问。需要配置端口转发。

# 方法一：端口转发

WSL2会动态分配IP地址，需要通过Windows主机转发端口。

# 在PowerShell（管理员）中执行
# 转发22端口（SSH）
netsh interface portproxy add v4tov4 listenport=22 listenaddress=0.0.0.0 connectport=22 connectaddress=<WSL2_IP> protocol=tcp

# 转发8091端口（Web服务）
netsh interface portproxy add v4tov4 listenport=8091 listenaddress=0.0.0.0 connectport=8091 connectaddress=<WSL2_IP> protocol=tcp

# 查看转发规则
netsh interface portproxy show all

# 删除转发规则
netsh interface portproxy delete v4tov4 listenport=22 listenaddress=0.0.0.0

获取WSL2 IP地址：

# 在WSL2中执行
ip addr show eth0 | grep inet | awk '{print $2}' | cut -d/ -f1

注意： WSL2重启后IP可能变化，需要重新配置转发。

# 方法二：镜像网络模式（推荐）

Windows 11或Windows 10新版本支持镜像网络模式，WSL2直接使用主机网络。

编辑C:\Users\<用户名>\.wslconfig：

[experimental]
autoMemoryReclaim=gradual
networkingMode=mirrored
dnsTunneling=true
firewall=true
autoProxy=true
sparseVhd=true
hostAddressLoopback=true

重启WSL：

wsl --shutdown

优点：

  • WSL2使用主机IP，无需端口转发
  • 自动配置DNS和防火墙
  • 支持IPv6

# 开放Windows防火墙

配置端口转发后还需要在防火墙中开放端口：

# 开放22端口
New-NetFirewallRule -DisplayName "WSL SSH" -Direction Inbound -LocalPort 22 -Protocol TCP -Action Allow

# 开放8091端口
New-NetFirewallRule -DisplayName "WSL Web" -Direction Inbound -LocalPort 8091 -Protocol TCP -Action Allow

或通过图形界面：

  1. 打开"高级安全Windows防火墙"
  2. 入站规则 → 新建规则
  3. 选择"端口" → 特定本地端口
  4. 输入端口号并允许连接

# WSL配置优化

# 限制资源使用

编辑C:\Users\<用户名>\.wslconfig：

[wsl2]
# 限制内存为8GB
memory=8GB

# 限制CPU核心数为4
processors=4

# 限制swap空间
swap=2GB

# 关闭页面文件
pageReporting=false

# 设置虚拟磁盘最大值
[experimental]
sparseVhd=true

重启WSL生效：

wsl --shutdown

# Docker集成

WSL2可以作为Docker Desktop的后端：

  1. 安装Docker Desktop
  2. 设置 → Resources → WSL Integration
  3. 启用需要的WSL发行版

优势：

  • Docker容器运行在WSL2中，性能更好
  • 文件系统访问速度快
  • 与Linux环境无缝集成

参考：WSL2和Docker安装教程 (opens new window)

# 常用命令

# WSL管理

# 列出所有发行版
wsl --list --all

# 列出正在运行的发行版
wsl --list --running

# 启动特定发行版
wsl -d Ubuntu-20.04

# 关闭特定发行版
wsl --terminate Ubuntu-20.04

# 关闭所有WSL
wsl --shutdown

# 设置默认发行版
wsl --set-default Ubuntu-20.04

# 转换为WSL2
wsl --set-version Ubuntu-20.04 2

# 卸载发行版
wsl --unregister Ubuntu-20.04

# 文件访问

Windows访问WSL文件：

\\wsl$\Ubuntu-20.04\home\username

WSL访问Windows文件：

cd /mnt/c/Users/username/Documents

# 性能优化

# 使用WSL2文件系统

文件放在WSL2文件系统中（/home）比放在Windows文件系统（/mnt/c）中性能更好。

推荐做法：

  • 开发项目放在~/projects
  • 需要Windows访问的文件放在/mnt/c

# 启用systemd

编辑/etc/wsl.conf：

[boot]
systemd=true

重启WSL生效。

# 常见问题

# WSL2磁盘空间不释放

即使删除文件，虚拟磁盘文件（ext4.vhdx）也不会自动缩小。

解决方法：

# 关闭WSL
wsl --shutdown

# 打开PowerShell（管理员）
diskpart

# 在diskpart中执行
select vdisk file="C:\Users\<用户名>\AppData\Local\Packages\...\ext4.vhdx"
compact vdisk
detach vdisk
exit

# 网络连接问题

# 检查DNS配置
cat /etc/resolv.conf

# 手动设置DNS
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf

# 权限问题

# WSL中访问Windows文件时权限全是777
# 可以在/etc/wsl.conf中配置
[automount]
options = "metadata,umask=22,fmask=11"
