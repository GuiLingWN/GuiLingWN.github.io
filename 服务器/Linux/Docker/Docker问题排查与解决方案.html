# Docker问题排查与解决方案

在使用Docker的过程中，经常会遇到各种问题。本文整理了一些常见问题的排查思路和解决方案。

# 磁盘空间问题

# 磁盘占用过高

Docker长期使用后会积累大量未使用的镜像、容器和网络资源，可以使用以下命令清理：

# 清理未使用的镜像、容器、网络和数据卷
docker system prune -a

注意： 这个命令会删除所有未被使用的资源，包括未使用的数据卷，执行前请确认重要数据已备份。

# Windows Docker Desktop磁盘空间不释放

在Windows环境下使用Docker Desktop时，即使删除了容器中的文件，ext4.vhdx文件占用的空间也不会自动释放。

# 解决方法一：使用optimize-vhd（推荐）

  1. 退出Docker Desktop
  2. 进入虚拟磁盘目录：
    cd C:\Users\PC\AppData\Local\Docker\wsl\data
  3. 关闭WSL：
    wsl --shutdown
  4. 优化虚拟磁盘：
    optimize-vhd -Path .\ext4.vhdx -Mode full
  5. 等待执行完成后重启Docker Desktop

# 解决方法二：使用diskpart压缩

  1. 关闭Docker Desktop，确认进程完全退出
  2. 在PowerShell中执行：
    # 关闭WSL2
wsl --shutdown

# 启动DiskPart
diskpart
  3. 在DiskPart命令窗口中执行：
    # 选择虚拟磁盘文件（路径根据实际情况调整）
select vdisk file="C:\Users\PC\AppData\Local\Docker\wsl\data\ext4.vhdx"

# 压缩文件
compact vdisk

# 卸载磁盘
detach vdisk

虚拟磁盘默认路径：

C:\Users\<用户名>\AppData\Local\Docker\wsl\data\ext4.vhdx

# 权限问题

# 容器内权限不足

进入容器时如果遇到权限问题，可以使用root用户身份进入：

docker exec -it -u root <container_name> /bin/bash

# 时区问题

# 容器时区不正确

容器默认使用UTC时区，可能导致日志时间显示不正确。解决方法：

方法一：在容器内修改软链接

ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

方法二：启动容器时挂载时区配置

docker run -v /etc/localtime:/etc/localtime:ro ...

方法三：设置环境变量

docker run -e TZ="Asia/Shanghai" ...

# 服务连接问题

# Docker守护进程无法连接

错误信息：

Cannot connect to the Docker daemon at unix:///var/run/docker.sock. 
Is the docker daemon running?

这个问题通常发生在断网、断电导致Docker未正常退出的情况下。

解决方法：

# 重启Docker服务
sudo service docker restart

如果是使用systemd管理的系统：

sudo systemctl restart docker

# GPU支持问题

# 安装nvidia-docker2失败

错误信息：Unable to locate package nvidia-docker2

解决方法：

# 添加NVIDIA Docker的GPG密钥
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | \
  sudo apt-key add -

# 获取发行版信息并添加仓库
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | \
  sudo tee /etc/apt/sources.list.d/nvidia-docker.list

# 更新软件包列表
sudo apt-get update

# 安装nvidia-docker2
sudo apt-get install nvidia-docker2

# 日志管理

# 限制容器日志大小

容器日志如果不加限制会不断增长，占用大量磁盘空间。启动容器时可以配置日志限制：

docker run \
  --log-opt max-size=1m \
  --log-opt max-file=5 \
  ...

参数说明：

  • max-size: 单个日志文件的最大大小
  • max-file: 保留的日志文件数量

# 其他实用技巧

# 查看容器运行状态

# 查看所有容器
docker ps -a

# 查看容器日志
docker logs <container_name>

# 实时查看日志
docker logs -f <container_name>

# 查看容器资源占用
docker stats

# 容器网络诊断

# 查看容器IP地址
docker inspect <container_name> | grep IPAddress

# 查看端口映射
docker port <container_name>

# 测试容器网络连通性
docker exec <container_name> ping <target>

# 预防措施

  1. 定期清理资源： 建立定期清理未使用资源的习惯
  2. 限制日志大小： 启动容器时配置日志限制
  3. 监控磁盘空间： 使用监控工具及时发现空间问题
  4. 正确关闭服务： 避免强制断电导致Docker异常
  5. 数据持久化： 重要数据使用数据卷持久化
