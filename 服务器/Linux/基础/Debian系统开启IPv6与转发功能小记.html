# Debian 系统开启 IPv6 与转发功能小记

在服务器管理中，随着 IPv6 的普及，我们经常需要对网络进行相应配置。这篇文章将分享一下在 Debian 系统上如何启用 IPv6 和 IP 转发功能，并记录几个实用的网络命令，希望能为有需要的朋友提供一些参考。

# 启用 IPv6

在某些系统或网络环境下，IPv6 可能被默认禁用了。如果需要启用它，我们可以通过修改内核参数来实现。

首先，检查或修改 /etc/sysctl.conf 配置文件，确保以下两项参数被设置为 0：

net.ipv6.conf.all.disable_ipv6 = 0
net.ipv6.conf.default.disable_ipv6 = 0
  • net.ipv6.conf.all.disable_ipv6 = 0：为所有接口启用 IPv6。
  • net.ipv6.conf.default.disable_ipv6 = 0：为默认接口启用 IPv6。

修改配置文件后，执行以下命令让配置立即生效：

sysctl -p

我们也可以使用 sysctl 命令直接查看当前的参数值，例如：

sysctl net.ipv6.conf.all.disable_ipv6
sysctl net.ipv6.conf.default.disable_ipv6

# 开启 IP 转发

如果希望你的服务器能像路由器一样转发网络数据包（例如，在搭建 VPN 或虚拟网络环境时），就需要开启 IP 转发功能。

同样，在 /etc/sysctl.conf 文件中添加或修改以下配置：

# 开启 IPv4 转发
net.ipv4.ip_forward = 1

# 开启 IPv6 转发
net.ipv6.conf.all.forwarding = 1

保存后，别忘了执行 sysctl -p 使之生效。

# 常用网络命令

这里也记录两个在配置过程中可能会用到的命令。

  1. 查看所有网卡及 IP 地址

    这个命令可以列出系统上所有网络接口的详细信息，包括 IPv4 和 IPv6 地址。

    ip address list
# 或者简写为
ip addr

  2. 获取访问外部地址时使用的源 IPv6 地址

    这是一个很有用的小技巧，可以通过 ip route 命令模拟一次路由查询，从而得知访问某个特定目的地址时，系统会使用哪个源 IP 地址。

    # 比如，获取访问 240c::6666 时所使用的源 IPv6 地址
ip route get 240c::6666 | awk -F'src ' '{print $2}' | cut -d' ' -f1

# 总结

通过简单修改 sysctl.conf 配置文件，我们就可以方便地在 Debian 系统上控制 IPv6 和 IP 转发的开关。这些基础网络配置虽然简单，却是许多高级网络应用的前提。希望这篇简短的笔记能对你有所帮助。
