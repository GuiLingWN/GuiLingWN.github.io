# Ubuntu 服务器运维实用笔记

在使用 Ubuntu 服务器的过程中，我积累了不少零散的命令和配置技巧。为了方便日后查阅，也为了与大家分享，我将这些笔记整理成文。内容涵盖了文件处理、网络配置、常用服务安装等多个方面，希望能成为你运维工具箱里的一份实用备忘。

# 一、文件与压缩

打包和解压是日常操作的必备技能。

# tar 命令 (tarballs, .tar.gz)

  • 压缩：将文件或目录打包并用 gzip 压缩。

    # -z: 使用 gzip 压缩
# -c: 创建归档
# -v: 显示过程
# -f: 指定文件名
tar -zcvf archive_name.tar.gz file_or_directory_to_compress

  • 解压缩：

    # -x: 解开归档
tar -zxvf archive_name.tar.gz

# zip 命令

  • 压缩：

    # 压缩单个文件或空目录
zip archive_name.zip file_to_compress

# 递归压缩目录及其所有内容
zip -r archive_name.zip directory_to_compress

  • 解压缩：

    # 解压到当前目录
unzip archive_name.zip

# 解压到指定目录
unzip -d /path/to/destination archive_name.zip

# 二、网络与防火墙

# 使用 iptables 开放端口

iptables 是 Linux 内核集成的强大防火墙工具。以下是如何用它来开放端口。

  1. 安装 iptables (如果尚未安装)：

    sudo apt install iptables

  2. 添加规则以开放端口 (例如 80 端口)：

    # -I INPUT: 在 INPUT 链的开头插入规则
# -p tcp: 协议为 TCP
# --dport 80: 目标端口为 80
# -j ACCEPT: 接受数据包
sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT

  3. 持久化规则： 上述规则在服务器重启后会失效。为了永久保存，需要安装 iptables-persistent。

    sudo apt install iptables-persistent

    安装过程中会询问是否保存当前的 IPv4 和 IPv6 规则，选择 “Yes”。

    之后，每当修改了 iptables 规则，运行以下命令即可保存：

    sudo netfilter-persistent save

# 网络诊断工具

  • 测试端口连通性 telnet：

    # 测试能否连接到 192.168.31.105 的 22 端口
telnet 192.168.31.105 22

  • 查看端口占用 netstat 或 lsof：

    # 使用 netstat 查看所有监听中的 TCP/UDP 端口及其进程
sudo netstat -tulnp

# 使用 lsof 查看指定端口的占用情况
sudo lsof -i :<port_number>

  • 安装 ifconfig: 新版的 Ubuntu 可能默认不带 ifconfig，可以通过安装 net-tools 来获取。

    sudo apt-get install net-tools

# 三、常用服务安装与管理

# SSH 服务

  • 安装与管理：

    # 检查 SSH 服务状态
ps -e | grep ssh

# 安装 SSH 服务端
sudo apt-get install openssh-server

# 启动服务
sudo service ssh start

  • 允许 root 用户登录： 出于安全考虑，不建议在公网服务器上开启。但在特定内网或测试环境需要时：

    1. 编辑 /etc/ssh/sshd_config。
    2. 将 #PermitRootLogin prohibit-password 修改或添加为 PermitRootLogin yes。
    3. 同时确保 PasswordAuthentication yes 是开启的（如果希望用密码登录）。
    4. 重启 SSH 服务：sudo service ssh restart。

# Nginx

从源码编译安装 Nginx 提供了更高的定制性。

# 下载源码并解压
wget http://nginx.org/download/nginx-1.19.4.tar.gz
tar -zxvf nginx-1.19.4.tar.gz
cd nginx-1.19.4/

# 配置安装目录
./configure --prefix=/usr/local/nginx

# 编译并安装
make && sudo make install

# 启动 Nginx
/usr/local/nginx/sbin/nginx

# (可选) 创建软链接，方便全局调用
ln -s /usr/local/nginx/sbin/nginx /usr/bin/nginx

# Redis

# 安装 Redis 服务器
sudo apt update
sudo apt install redis-server

配置远程访问：

  1. 编辑配置文件 sudo vi /etc/redis/redis.conf。
  2. 将 bind 127.0.0.1 注释掉（在行首加 #），或改为 bind 0.0.0.0。
  3. 将 protected-mode yes 改为 protected-mode no。
  4. 重启 Redis 服务：sudo service redis-server restart。

注意：开放 Redis 到公网非常危险，请确保设置了强密码并有其他防火墙规则保护。

# MySQL

  • 通过 apt 安装后，默认的 root 密码信息可能存储在 /etc/mysql/debian.cnf 中。
  • 为 root 用户设置新密码 (MySQL 8.0+)：
    -- 登录 MySQL
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'YourNewPassword';
FLUSH PRIVILEGES;
  • 允许远程访问：
    GRANT ALL ON *.* TO 'root'@'%' IDENTIFIED BY 'YourNewPassword' WITH GRANT OPTION;
FLUSH PRIVILEGES;

# 四、系统管理与故障排除

  • 设置 root 密码： 新安装的 Ubuntu Server 可能没有为 root 用户设置密码，可以通过当前用户执行：

    sudo passwd root

  • 进程管理：

    # 查看所有与 node 相关的进程
ps aux | grep node

# 结束指定 PID 的进程
kill 1234

  • 解决 vi 编辑器方向键变 ABCD： 这是因为安装了不完整的 vim 版本。重新安装完整版即可。

    sudo apt-get remove vim-common
sudo apt-get install vim

  • 解决 dpkg frontend lock 错误： 当有其他 apt 进程正在运行时，会遇到此错误。如果确定没有其他 apt 在运行，可能是上次异常退出导致的锁文件残留。

    # 强制删除锁文件（请谨慎使用）
sudo rm /var/lib/dpkg/lock

  • 定时任务 crontab：

    # 查看当前用户的定时任务
crontab -l

# 编辑定时任务
crontab -e

# 删除所有定时任务
crontab -r
