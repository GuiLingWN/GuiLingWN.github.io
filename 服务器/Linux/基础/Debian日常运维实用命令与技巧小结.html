# Debian 日常运维实用命令与技巧小结

Debian 以其稳定和强大而著称，是许多服务器的首选操作系统。在日常使用和维护过程中，我积累了一些常用的命令和解决问题的小技巧，内容涵盖了磁盘管理、网络配置、软件包处理等方面。在这里，我将这些零散的笔记整理成一篇技术分享，希望能为正在使用 Debian 的朋友们提供一些便利。

# 一、磁盘与文件管理

管理服务器时，我们经常需要关注磁盘空间和文件。以下是一些实用的命令。

# 查看磁盘和目录大小

  • 查看整体磁盘占用情况：

    df -Th

    这个命令会以易于阅读的格式（-h）并显示文件系统类型（-T）来列出所有挂载点。

  • 查看指定目录的大小：

    # 查看当前目录下每个子目录的大小，并按大小排序
du -h --max-depth=1 | sort -rh

# 仅查看指定目录的总大小
du -sh [目录名]

# 使用 find 命令进行高级文件管理

find 是一个非常强大的工具，尤其适合批量处理文件。

  • 统计目录下的文件数量：

    find /path/to/your/dir -type f | wc -l

  • 查找并处理旧文件：

    # 列出目录下超过一年的文件
find /path/to/your/dir -type f -mtime +365

# 删除目录下超过一年的文件（请谨慎操作！）
find /path/to/your/dir -type f -mtime +365 -exec rm {} \;

# 移动目录下超过200天的文件到备份目录
find /path/to/your/dir -type f -mtime +200 -exec mv {} /path/to/backup/ \;

  • 统计旧文件的总大小：

    # 统计目录下100天前的文件总大小（单位：GB）
find /path/to/your/dir -type f -mtime +100 -exec du -b {} + | awk '{ total += $1 } END { print total/1024/1024/1024 }'

# 二、软件包管理 (APT)

apt 是 Debian 的核心工具之一。更换国内镜像源可以大大提升下载速度。

# 更换 APT 软件源

  1. 查看你的 Debian 版本代号：

    cat /etc/os-release

    记下版本代号，例如 bullseye (Debian 11) 或 bookworm (Debian 12)。

  2. 备份当前的源列表：

    cp /etc/apt/sources.list /etc/apt/sources.list.bak

  3. 编辑源文件 (/etc/apt/sources.list)，将其内容替换为国内镜像源，例如阿里云的源。

    Debian 11 (bullseye) 示例：

    deb http://mirrors.aliyun.com/debian/ bullseye main non-free contrib
deb-src http://mirrors.aliyun.com/debian/ bullseye main non-free contrib

deb http://mirrors.aliyun.com/debian-security/ bullseye-security main non-free contrib
deb-src http://mirrors.aliyun.com/debian-security/ bullseye-security main non-free contrib

deb http://mirrors.aliyun.com/debian/ bullseye-updates main non-free contrib
deb-src http://mirrors.aliyun.com/debian/ bullseye-updates main non-free contrib

  4. 更新软件包列表：

    apt update

# 解决 APT 安装问题

  • 缺少 GPG 密钥： 如果在 apt update 时遇到 gnupg 相关的错误，通常是缺少签名工具。

    apt-get update
apt-get install -y gnupg

  • dpkg 返回错误码 (1)： 当安装或卸载软件中断时，可能会遇到这个棘手的问题。可以尝试以下修复步骤：

    cd /var/lib/dpkg/
sudo mv info/ info_bak          # 备份 info 目录
sudo mkdir info                 # 新建空的 info 目录
sudo apt-get update             # 更新源
sudo apt-get -f install         # 尝试修复安装
sudo mv info/* info_bak/        # 将新生成的文件移回备份
sudo rm -rf info                # 删除空目录
sudo mv info_bak info           # 恢复原始 info 目录

# 三、网络配置与诊断

# 常用网络命令

  • 查看端口占用：

    netstat -ntulp
# -n: 数字形式显示地址和端口
# -t: 显示 TCP 端口
# -u: 显示 UDP 端口
# -l: 显示监听中的端口
# -p: 显示占用端口的进程

  • 测试 UDP 端口连通性： 可以使用 ncat (需要 apt install ncat 安装)。

    # -z: 不发送数据，仅报告状态
# -v: 显示详细信息
# -u: 使用 UDP 协议
nc -zvu <目标IP> <端口>

  • 查看和修改网关：

    # 查看当前路由和网关
ip route show

# 临时修改默认网关（重启后失效）
ip route del default via <旧网关IP>
ip route add default via <新网关IP> dev <网卡名>

    永久修改需要编辑 /etc/network/interfaces 文件。

# SSH 服务配置

  • 安装 SSH 服务：

    apt install -y openssh-server

  • 允许 root 用户登录： 出于安全考虑，不推荐在公网上这样做。但在内网或特定测试环境中，可以修改配置文件：

    1. 编辑 /etc/ssh/sshd_config。
    2. 找到 PermitRootLogin 这一行，修改为 yes。
    3. 重启 SSH 服务：service ssh restart。

# 四、系统服务与日志

# 查看 Cron 定时任务日志

Debian 默认不记录 Cron 日志。要开启它：

  1. 编辑 /etc/rsyslog.conf。
  2. 找到 #cron.* /var/log/cron.log 这一行，去掉前面的 # 注释符。
  3. 重启日志服务：/etc/init.d/rsyslog restart。

之后，Cron 的执行记录就会出现在 /var/log/cron.log 文件中了。

# 时区问题

  • 设置系统时区： 使用 tzconfig 命令或 dpkg-reconfigure tzdata 可以交互式地设置时区。

  • 解决 Cron 时差问题： 如果发现定时任务的执行时间不对，可能是因为 /etc/localtime 文件不正确。可以强制将其链接到正确的时区文件。

    # 将时区设置为上海时间
rm /etc/localtime
ln -s /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
