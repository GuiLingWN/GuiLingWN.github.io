# 精细化控制 SSH 访问：巧用 sshd_config 中的 AllowUsers

在管理 Linux 服务器时，安全性是重中之重。SSH 作为我们远程访问服务器的主要入口，其安全策略直接关系到整个系统的安危。除了使用密钥认证、修改默认端口等常规操作外，我们还可以通过配置 sshd_config 文件中的 AllowUsers 指令，实现对登录用户的白名单控制，从而进一步加固我们的服务器。

# 什么是 AllowUsers？

AllowUsers 是 OpenSSH 服务器配置文件（通常位于 /etc/ssh/sshd_config）中的一个指令。它的作用非常直接：定义一个允许通过 SSH 登录到本机的用户白名单。

一旦设置了这个指令，只有在名单上的用户才能成功登录，其他所有用户（无论密码或密钥是否正确）都将被拒绝访问。这是一种非常有效的访问控制手段，特别适用于那些用户账号固定、不希望随意创建新用户就能登录的生产环境。

# 如何配置 AllowUsers

配置过程非常简单，只需三步：编辑配置、添加用户、重启服务。

# 1. 编辑 sshd_config 文件

使用你喜欢的文本编辑器打开 SSH 配置文件：

sudo nano /etc/ssh/sshd_config

# 2. 添加 AllowUsers 指令

在文件的任意位置（通常建议放在认证相关的配置区域），添加 AllowUsers 指令，后面跟上允许登录的用户名，用户名之间用空格隔开。

示例：

假设你只希望 root、user1 和 user2 这三个用户能够登录服务器，你可以这样配置：

AllowUsers root user1 user2

更高级的用法：

AllowUsers 还支持 user@host 的格式，用于限制特定用户只能从特定的 IP 或主机名登录。

# 只允许 user1 从 192.168.1.100 登录
AllowUsers user1@192.168.1.100

# 允许使用通配符
# 允许所有来自 .example.com 域的用户登录
AllowUsers *@*.example.com

# 3. 重启 SSH 服务

修改配置后，必须重启 sshd 服务才能使其生效。

# 对于使用 systemd 的系统 (如 Debian, Ubuntu, CentOS 7+)
sudo systemctl restart sshd

# 对于使用 init.d 的旧系统
sudo /etc/init.d/sshd restart

重启之后，新的访问规则就立即生效了。此时，如果你尝试用一个未在 AllowUsers 列表中的用户登录，即便密码正确，也会被服务器拒绝。

# Allow/Deny 指令的处理顺序

值得注意的是，sshd 服务中有一套完整的用户和组的访问控制指令，它们的处理顺序是固定的：

  1. DenyUsers （拒绝用户列表）
  2. AllowUsers （允许用户列表）
  3. DenyGroups （拒绝组列表）
  4. AllowGroups （允许组列表）

sshd 会按这个顺序依次检查。例如，如果一个用户同时出现在 DenyUsers 和 AllowUsers 中，因为 DenyUsers 先被处理，所以该用户最终会被拒绝登录。

# 总结

通过配置 AllowUsers，我们为服务器的 SSH 入口增加了一道坚固的“白名单”防火墙。这个看似简单的配置，却能有效地防止未经授权的账户访问系统，极大地提升了服务器的安全性。在对服务器权限有严格要求的场景下，这是一个非常值得推荐的安全实践。
