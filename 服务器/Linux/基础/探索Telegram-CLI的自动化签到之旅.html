# 探索 Telegram-CLI 的自动化签到之旅

自动化总是能激起技术爱好者的兴趣，它能将我们从重复性的劳动中解放出来。最近，我尝试利用 telegram-cli 这个强大的命令行工具，在我的 Linux 服务器上实现了一个 Telegram 频道的自动签到功能。整个过程虽然遇到了一些小坑，但最终成功运行。在这里，我将整个探索过程记录下来，希望能为有类似需求的朋友提供一份参考。

# 什么是 telegram-cli？

telegram-cli 是一个非官方的 Telegram 客户端，它允许我们通过命令行与 Telegram 进行交互，比如发送消息、接收消息、管理频道等。正是由于其命令行的特性，它非常适合被集成到各种自动化脚本中。

# 方案一：使用 Snap 快速安装（最简单）

对于想快速上手的朋友，使用 snap 是最便捷的安装方式。它免去了编译的烦恼，几条命令就能搞定。

# 1. 安装 snapd
apt update
apt install snapd

# 2. 安装 core snap
snap install core

# 3. 安装 telegram-cli
snap install telegram-cli

安装完成后，你就可以直接在命令行使用 telegram-cli 了。这个方案虽然简单，但对于需要深度定制或在某些特定环境下，可能不是最佳选择。

# 方案二：从源码编译安装（更灵活）

对于追求灵活性和稳定性的用户来说，从源码编译安装是更好的选择。这能确保我们使用的是最新的代码，并且能更好地适应我们的系统环境。

# 1. 准备工作：克隆源码与安装依赖

原版的 telegram-cli 仓库可能存在编译问题，这里我选择了一个持续维护的 fork 版本 kenorb-contrib。

# 克隆源码仓库
git clone --recursive https://github.com/kenorb-contrib/tg.git
cd tg

接下来，根据你的操作系统安装编译所需的依赖包。

对于 Debian/Ubuntu 系统：

sudo apt-get update
sudo apt-get install libreadline-dev libconfig-dev libssl-dev lua5.2 liblua5.2-dev libevent-dev libjansson-dev libpython-dev make zlib1g-dev

注意：如果在 ./configure 步骤中遇到 zlib error，通常是因为缺少 zlib1g-dev 这个包，安装即可。

对于 CentOS 7 系统：

yum -y install lua-devel openssl-devel libconfig-devel readline-devel libevent-devel jansson-devel python-devel gcc

# 2. 编译与安装

依赖安装完毕后，就可以开始编译了。

./configure
make

编译成功后，telegram-cli 的可执行文件会生成在 bin/ 目录下。

# 3. 首次登录与测试

在正式用于自动化之前，我们需要手动运行一次以完成登录授权。

# 运行 telegram-cli，-k 参数指定公钥位置
/path/to/your/tg/bin/telegram-cli -k /path/to/your/tg/tg-server.pub

首次运行会提示你输入手机号、接收验证码，并可能需要输入二次验证密码。成功登录后，会在 ~/.telegram-cli 目录下生成会话文件，这样下次运行时就无需再次授权了。

# 实现自动化签到

有了可用的 telegram-cli，我们就可以编写自动化脚本了。这里我分享两种方式：纯 Shell 脚本和 Node.js 脚本。

# 方式一：使用 Shell 脚本与 Crontab

这是最直接的方式。我们可以创建一个 Shell 脚本，利用 telegram-cli 的 -e 参数直接执行命令并退出。

  1. 创建 autotg.sh 脚本：

    #!/bin/bash
# 使用 -W 选项等待操作完成，-e 选项执行命令后退出
/path/to/your/tg/bin/telegram-cli -W -e "msg <频道或联系人名称> <签到指令>"

    例如，向名为“飞云小助手”的 Bot 发送 /checkin 指令： telegram-cli -W -e "msg 飞云小助手 /checkin"

  2. 为脚本添加执行权限：

    chmod +x autotg.sh

  3. 添加到 Crontab 定时任务：

    crontab -e

    在打开的编辑器中添加一行，设置每天的特定时间执行脚本。

    # 每天凌晨 2:30 执行签到
30 2 * * * bash /path/to/your/autotg.sh

# 方式二：使用 Node.js 脚本（带随机延迟）

为了让签到行为看起来不那么像机器人，我更喜欢在脚本中加入一个随机延迟。Node.js 非常适合做这件事。

  1. 创建 autotg.js 脚本：

    const { exec } = require('child_process');

(async () => {
  // 设置一个最多3分钟的随机延迟
  const delay = Math.random() * 1000 * 60 * 3;
  console.log(`[${new Date().toLocaleString()}] 等待 ${Math.floor(delay / 1000)} 秒后执行...`);
  await sleep(delay);

  // **重要提示**：这里的 telegram-cli 路径必须是绝对路径，否则可能因环境变量问题而找不到命令
  const command = '/path/to/your/tg/bin/telegram-cli -W -e "msg 飞云小助手 /checkin"';
  console.log(`[${new Date().toLocaleString()}] 执行命令: ${command}`);
  await executeCommand(command);
  console.log(`[${new Date().toLocaleString()}] 执行完毕。`);
})();

function executeCommand(command) {
  return new Promise((resolve, reject) => {
    exec(command, (error, stdout, stderr) => {
      if (error) console.error(`执行出错: ${error}`);
      if (stdout) console.log(`输出: ${stdout}`);
      if (stderr) console.error(`错误输出: ${stderr}`);
      resolve();
    });
  });
}

function sleep(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

  2. 添加到 Crontab：

    # 每天 17:33 执行 Node.js 脚本，并将日志输出到文件
33 17 * * * node /path/to/your/autotg.js >> /path/to/logs/autotg.log 2>&1

# 遇到的坑

在整个过程中，最大的一个坑就是在脚本中调用 telegram-cli 时必须使用绝对路径。因为 Crontab 执行环境的 PATH 变量非常有限，如果不指定绝对路径，系统很可能找不到 telegram-cli 这个命令，导致脚本静默失败，没有任何反应。

# 总结

通过 telegram-cli，我们成功地将一个日常的手动操作转化为了无人值守的自动化任务。无论是简单的 Shell 脚本，还是功能更丰富的 Node.js 脚本，都为我们打开了通往更多 Telegram 自动化玩法的大门。希望这次的分享能给你带来启发！
