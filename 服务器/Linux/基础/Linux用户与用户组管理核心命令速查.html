# Linux 用户与用户组管理核心命令速查

作为 Linux 系统管理员，用户和用户组的管理是绕不开的基础操作。无论是为新同事创建账号、分配权限，还是进行安全加固，我们都需要与一系列命令打交道。这篇文章是我个人整理的一份备忘录，汇总了在日常工作中常用的用户与用户组管理命令，希望能帮助大家快速查找和回顾。

# 一、用户管理

# 1. 创建新用户

在 Linux 中，创建用户通常有两种方式：adduser 和 useradd。

  • adduser (推荐)：这是一个交互式的脚本，使用起来非常友好。它会在创建用户的同时，自动创建家目录、设置密码，并引导你填写一些额外信息。

    # 新建一个名为 phpq 的用户
adduser phpq

  • useradd：这是一个更底层的命令，适合在脚本中使用，可以配合参数一次性完成配置。

    # 新建 phpq 用户并将其主工作组设置为已存在的 test 组
useradd -g test phpq

# 为新用户设置密码
passwd phpq

    useradd 的常用参数：

    • -g：指定用户所属的主工作组。
    • -d：指定用户的家目录。
    • -s：指定用户登录后使用的 Shell。

# 2. 修改用户信息

最常见的修改操作是为用户添加附加工作组。

  • usermod：这是修改用户属性的强大工具。

    # 将用户 username 添加到 groupname 组中，同时保留其原有的附加组
usermod -aG groupname username

    注意：这里的 -a（append）参数至关重要。如果只使用 usermod -G groupname username，该用户会从其他所有附加组中被移除，只保留 groupname 这个组，这通常不是我们想要的结果。

  • gpasswd：这是专门管理用户组的命令，也可以用来向组中添加成员。

    # 同样是将用户 user 添加到 group 组中
gpasswd -a user group

# 3. 临时锁定与解锁用户

如果希望临时禁止某个用户登录，而不是直接删除，可以使用 passwd 命令进行锁定。

# 锁定 peter 用户的账号
passwd -l peter

# 解锁 peter 用户的账号
passwd -u peter

小技巧：其原理是在 /etc/shadow 文件中对应用户的加密密码串前添加一个 ! 或 *。手动编辑虽然也能实现，但不如使用 passwd 命令来得安全方便。

# 4. 永久删除用户

当用户确定不再需要时，可以使用 userdel 命令进行删除。

# 仅删除用户 peter，但保留其家目录
userdel peter

# 删除用户的同时，也删除其家目录和邮件池（-r 代表 --remove）
userdel -r peter

# 二、用户组管理

# 1. 创建与删除用户组

# 新建一个名为 test 的工作组
groupadd test

# 删除 test 工作组
groupdel test

# 2. 从用户组中删除用户

可以使用 gpasswd 命令的 -d 选项。

# 从 group 组中移除用户 user
gpasswd -d user group

另外一种方式是直接编辑 /etc/group 文件，找到对应的组，从成员列表中删除该用户名即可。但同样，直接使用命令是更推荐的做法。

# 三、查看用户信息

  • id 命令：显示指定用户的 UID（用户ID）、GID（主组ID）以及所属的所有用户组。

    id username

  • 查看密码文件：/etc/passwd 文件存储了系统所有用户的信息（但不含密码）。

    cat /etc/passwd

# 四、Sudo 权限配置

有时我们希望某个组的用户都能免密切换到另一个公共账号下执行操作。这可以通过配置 sudoers 实现。

一个好的实践是在 /etc/sudoers.d/ 目录下创建一个新的配置文件（例如，touch /etc/sudoers.d/my-sudo-rules），而不是直接修改主文件 /etc/sudoers。

例如，允许 etl 组的所有成员都能免密码切换到 etl 用户：

# 文件：/etc/sudoers.d/etl-permissions

%etl ALL=(ALL) NOPASSWD: /bin/su etl
%etl ALL=(ALL) NOPASSWD: /bin/su - etl

这里的 %etl 表示 etl 用户组。这条规则意味着，etl 组的任何成员在任何主机上（第一个 ALL）都可以作为任何用户（第二个 ALL）免密码（NOPASSWD）执行后面的命令。

关于 sudo 的更详细配置，可以参考这篇详细的文章：Sudoers 文件配置详解 (opens new window)
