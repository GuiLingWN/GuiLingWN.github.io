# Termux安卓终端配置与使用指南

Termux是Android平台上强大的终端模拟器，可以在手机上运行Linux环境。本文介绍如何配置Termux并实现SSH远程访问和Git代码管理。

# 初始配置

# 安装与换源

首次打开Termux后，建议先更换为国内镜像源以提升下载速度。

换源操作：

termux-change-repo

使用方向键和空格键选择镜像源，推荐选择清华或中科大镜像。

# 基础软件安装

# 更新软件包列表
apt update && apt upgrade

# 安装常用工具
apt install vim git wget curl

# SSH服务配置

# 启用SSH服务

Termux可以作为SSH服务器，允许其他设备通过SSH连接。

安装并配置：

# 安装OpenSSH
apt install openssh

# 设置登录密码
passwd

# 启动SSH服务
sshd

查看连接信息：

# 查看IP地址（手机和电脑需在同一局域网）
ifconfig

# Termux SSH默认端口为8022
# 连接命令示例：
# ssh -p 8022 <手机IP>

# 从电脑连接Termux

Windows系统（使用Windows终端或PowerShell）：

ssh -p 8022 192.168.0.8

Linux/Mac系统：

ssh -p 8022 user@192.168.0.8

输入之前设置的密码即可连接。

# 存储访问

# 授权存储权限

Termux需要获取存储权限才能访问手机文件系统。

# 请求存储权限
termux-setup-storage

执行后会弹出权限请求对话框，点击"允许"。

# 访问存储目录

授权后会在Termux home目录下创建storage目录：

# 进入存储目录
cd ~/storage

# 查看可用的快捷方式
ls -la ~/storage

常用目录：

  • ~/storage/shared: 内部存储根目录
  • ~/storage/downloads: 下载目录
  • ~/storage/dcim: 相机目录
  • ~/storage/music: 音乐目录

访问SD卡：

cd /sdcard
# 或
cd ~/storage/shared

# Git配置与使用

在Termux中可以完整使用Git进行代码管理。

# 安装Git

apt install git

# 配置SSH密钥

生成SSH密钥用于免密访问Git仓库：

# 生成SSH密钥对
ssh-keygen -t rsa -C "your_email@example.com"

# 查看公钥（复制添加到Git服务器）
cd ~/.ssh
cat id_rsa.pub

# 克隆仓库

# 使用HTTP协议
git clone http://192.168.2.5/username/repo.git

# 使用SSH协议
git clone git@192.168.2.5:username/repo.git

# 常见问题：dubious ownership

克隆到存储目录的仓库可能会遇到权限警告：

fatal: detected dubious ownership in repository at '/storage/emulated/0/git/repo'

解决方法：

git config --global --add safe.directory /storage/emulated/0/git/repo

# 基本操作

# 查看状态
git status

# 拉取更新
git pull

# 添加文件
git add .

# 提交
git commit -m "commit message"

# 推送
git push

# 实用场景

# 场景一：移动笔记管理

使用Termux + Git + Obsidian/Joplin管理个人笔记：

  1. 在Termux中克隆笔记仓库到存储目录
  2. 使用支持外部存储的笔记应用编辑
  3. 通过Termux进行Git同步
# 克隆笔记仓库
cd ~/storage/shared/Documents
git clone http://git-server/notes.git

# 同步脚本
cd ~/storage/shared/Documents/notes
git add .
git commit -m "mobile update"
git push

# 场景二：Web服务器

运行简单的Web服务：

# 安装Node.js
apt install nodejs

# 启动HTTP服务器
npx http-server -p 8080

# 或使用Python
python -m http.server 8080

# 场景三：数据库学习

# 安装SQLite
apt install sqlite

# 使用数据库
sqlite3 test.db

# 常用命令

# 包管理

# 搜索软件包
apt search <package>

# 安装软件包
apt install <package>

# 卸载软件包
apt remove <package>

# 列出已安装
apt list --installed

# 进程管理

# 查看进程
ps aux

# 结束进程
kill <pid>

# 后台运行
<command> &

# 使用screen保持会话
apt install screen
screen -S session_name

# 系统信息

# 查看系统信息
uname -a

# 查看磁盘空间
df -h

# 查看内存使用
free -h

# 优化建议

# 美化终端

# 安装zsh
apt install zsh

# 安装Oh My Zsh
sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# 配置别名

编辑~/.bashrc或~/.zshrc：

# 常用别名
alias ll='ls -lh'
alias la='ls -lah'
alias gs='git status'
alias gp='git pull'

# 启动脚本

创建~/.bashrc添加启动时自动执行的命令：

# 启动SSH服务
sshd

# 显示欢迎信息
echo "Welcome to Termux!"

# 注意事项

  1. 电池消耗： 后台运行服务会增加电池消耗
  2. 网络限制： 某些运营商可能限制端口访问
  3. 存储权限： 及时授予存储权限以便访问文件
  4. 系统更新： 定期更新软件包保持安全
  5. 数据备份： 重要数据及时备份，避免丢失
