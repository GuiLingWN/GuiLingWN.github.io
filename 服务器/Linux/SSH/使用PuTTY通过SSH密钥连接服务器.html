# 使用PuTTY通过SSH密钥连接服务器

SSH密钥认证是一种比密码更安全的服务器登录方式。本文记录了使用PuTTY工具通过SSH密钥连接远程服务器的配置过程。

# 什么是SSH密钥认证

SSH密钥认证使用一对密钥（公钥和私钥）来验证身份，相比密码认证有以下优势：

  • 更安全：不会因为密码泄露或被暴力破解而导致服务器被攻击
  • 更便捷：配置完成后无需每次输入密码
  • 可管理：可以为不同的设备生成不同的密钥对

# 生成SSH密钥对

# 使用PuTTYgen生成

  1. 打开PuTTYgen工具
  2. 选择密钥类型（推荐RSA，密钥长度2048或4096位）
  3. 点击"Generate"按钮，随机移动鼠标生成密钥
  4. 设置密钥密码（Key passphrase），增加额外的安全层
  5. 保存私钥（Save private key）到本地安全位置
  6. 复制公钥内容（Public key for pasting...）

# Linux/Mac系统生成

# 生成RSA密钥对
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 查看公钥内容
cat ~/.ssh/id_rsa.pub

# 配置服务器

# 添加公钥到服务器

  1. 登录到服务器
  2. 编辑authorized_keys文件：
    mkdir -p ~/.ssh
vim ~/.ssh/authorized_keys
  3. 将公钥内容粘贴到文件中（每个公钥占一行）
  4. 设置正确的权限：
    chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# 配置SSH服务

编辑SSH配置文件：

sudo vim /etc/ssh/sshd_config

确认以下配置：

# 启用公钥认证
PubkeyAuthentication yes

# 指定公钥文件位置
AuthorizedKeysFile .ssh/authorized_keys

# 可选：禁用密码登录（提高安全性）
PasswordAuthentication no

重启SSH服务：

# Debian/Ubuntu
sudo systemctl restart sshd

# CentOS/RHEL
sudo systemctl restart sshd

# 使用PuTTY连接

# 配置Session

  1. 打开PuTTY
  2. 在Session页面填写：
    • Host Name: 服务器IP地址或域名
    • Port: SSH端口（默认22）
    • Connection type: SSH

# 配置密钥

  1. 展开Connection → SSH → Auth
  2. 点击"Browse"按钮选择私钥文件（.ppk格式）
  3. 返回Session页面，保存配置（Save）

# 连接服务器

  1. 双击保存的Session或点击"Open"
  2. 首次连接会提示确认服务器指纹，点击"是"
  3. 如果私钥设置了密码，输入密钥密码
  4. 成功登录服务器

# 安全建议

# 密钥管理

  1. 私钥保护：

    • 私钥文件设置严格的访问权限
    • 为私钥设置强密码
    • 不要将私钥上传到云端或共享

  2. 公钥管理：

    • 定期审查authorized_keys文件
    • 删除不再使用的公钥
    • 为不同用途使用不同密钥对

# SSH服务加固

  1. 修改默认端口：

    Port 2222  # 改为非标准端口

  2. 限制登录用户：

    AllowUsers username1 username2

  3. 禁用root登录：

    PermitRootLogin no

  4. 启用防火墙：

    # 只允许特定IP访问SSH端口
sudo ufw allow from 1.2.3.4 to any port 22

# 常见问题

# 权限错误

如果连接失败，检查文件权限：

# .ssh目录必须是700
ls -ld ~/.ssh

# authorized_keys必须是600
ls -l ~/.ssh/authorized_keys

# 密钥格式转换

PuTTY使用.ppk格式，OpenSSH使用标准格式。

OpenSSH转PuTTY：

  1. 使用PuTTYgen导入OpenSSH私钥
  2. 保存为.ppk格式

PuTTY转OpenSSH：

  1. 使用PuTTYgen加载.ppk文件
  2. Conversions → Export OpenSSH key

# 连接超时

检查网络和防火墙配置：

# 测试端口连通性
telnet server-ip 22

# 查看SSH服务状态
sudo systemctl status sshd

# 检查防火墙规则
sudo ufw status

# 多密钥管理

# 配置SSH config

在~/.ssh/config中配置多个主机：

# 生产服务器
Host prod-server
    HostName 1.2.3.4
    User admin
    Port 2222
    IdentityFile ~/.ssh/prod_rsa

# 开发服务器
Host dev-server
    HostName 5.6.7.8
    User developer
    Port 22
    IdentityFile ~/.ssh/dev_rsa

使用时直接：

ssh prod-server
ssh dev-server
