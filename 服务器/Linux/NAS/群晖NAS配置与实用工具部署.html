# 群晖NAS配置与实用工具部署

在使用群晖NAS或黑群晖系统时，经常需要配置SSH访问、部署各种Docker服务。本文整理了一些实用的配置方法和工具部署指南。

# SSH访问配置

# 启用root登录

群晖DSM默认不允许使用root直接登录SSH，需要先使用普通用户登录再切换。

操作步骤：

  1. 使用普通用户SSH登录（如用户名wn）
  2. 切换到root用户：
    sudo -i
# 输入当前用户的密码

SSH访问信息示例：

  • 地址：192.168.11.2:22000
  • 用户：（根据实际情况设置）
  • 密码：（根据实际情况设置）

安全建议：

  • 不要开放SSH到公网
  • 定期更新密码
  • 考虑使用密钥登录替代密码

# WireGuard VPN服务

使用wg-easy可以快速部署WireGuard VPN服务，提供图形化管理界面。

生成密码哈希：

docker run -it ghcr.io/wg-easy/wg-easy wgpw 'your_password'

部署服务：

docker run -d \
  --name=wg-easy \
  -e WG_HOST=your.domain.com \
  -e PASSWORD_HASH='$2a$12$...' \
  -e WG_DEFAULT_ADDRESS=10.8.0.x \
  -e WG_DEFAULT_DNS=192.168.11.11 \
  -e WG_PORT=12580 \
  -v /volume2/docker/wg-easy:/etc/wireguard \
  -p 12580:12580/udp \
  -p 51821:51821/tcp \
  --cap-add=NET_ADMIN \
  --cap-add=SYS_MODULE \
  --sysctl="net.ipv4.conf.all.src_valid_mark=1" \
  --sysctl="net.ipv4.ip_forward=1" \
  --restart unless-stopped \
  ghcr.io/wg-easy/wg-easy

参数说明：

  • WG_HOST: 服务器公网域名或IP
  • PASSWORD_HASH: 管理界面登录密码的哈希值
  • WG_DEFAULT_ADDRESS: VPN客户端IP地址段
  • WG_DEFAULT_DNS: VPN客户端使用的DNS服务器
  • WG_PORT: WireGuard监听端口（UDP）
  • 管理界面端口：51821

访问管理界面：

http://nas-ip:51821

# Alist网盘管理

Alist是一个支持多种网盘的目录列表程序，可以方便地管理和分享文件。

部署命令：

docker run --name alist -d \
  --restart=unless-stopped \
  -v /volume2/docker/alist/data:/opt/alist/data \
  -v /volume2/docker/alist/root:/opt/alist/root \
  -v /volume2/迅雷:/volume2/迅雷 \
  -v /volume2/AList:/volume2/AList \
  -p 5244:5244 \
  -e PUID=0 \
  -e PGID=0 \
  -e UMASK=022 \
  -e "TZ=Asia/Shanghai" \
  xhofe/alist:v3.40.0

设置管理员密码：

docker exec -it alist ./alist admin set your_password

挂载目录说明：

  • /volume2/docker/alist/data: Alist数据目录
  • /volume2/迅雷: 迅雷下载目录（可选）
  • /volume2/AList: 网盘挂载目录（可选）

根据实际需求调整挂载路径。

# 阿里云盘WebDAV

将阿里云盘挂载为WebDAV协议，方便在各种设备上访问。

刷新登录Token：

  1. SSH登录NAS并切换到root：

    sudo -i

  2. 生成登录二维码：

    /var/packages/aliyundrive-webdav/target/bin/aliyundrive-webdav qr login

  3. 使用阿里云盘APP扫描二维码完成登录

注意： 阿里云盘token有效期有限，过期后需要重新登录刷新。

# PVE虚拟化配置

在Proxmox VE上安装黑群晖时，需要配置磁盘直通。

# 硬盘直通配置

添加物理硬盘到虚拟机：

# 查看硬盘设备ID
ls -l /dev/disk/by-id/

# 将硬盘添加到虚拟机（VM ID为108）
qm set 108 -sata1 /dev/disk/by-id/ata-SEAGATE_ST4000VX007_Z9LX2VPR

# 导入群晖镜像到虚拟机
qm importdisk 108 /var/lib/vz/template/iso/DSM_image.img local

参数说明：

  • qm set: 设置虚拟机配置
  • 108: 虚拟机ID
  • -sata1: SATA控制器编号
  • 最后的路径为物理硬盘的设备ID

查看硬盘信息：

# 查看所有硬盘
lsblk

# 查看硬盘详细信息
smartctl -a /dev/sda

# 注意事项

  1. 硬盘直通前的准备：

    • 确认硬盘没有被系统使用
    • 备份重要数据
    • 记录硬盘的by-id路径（更稳定）

  2. 群晖系统安装：

    • 使用对应型号的引导文件
    • 注意SN和MAC地址的修改
    • 首次安装完成后快照备份

  3. 性能优化：

    • 使用VirtIO驱动（如果支持）
    • 开启硬盘缓存（注意数据安全）
    • 合理分配CPU和内存

# 安全建议

  1. 网络安全：

    • 使用防火墙限制访问IP
    • 修改默认端口
    • 定期更新系统和软件包

  2. 数据安全：

    • 定期备份重要数据
    • 使用RAID阵列（如果有多块硬盘）
    • 考虑异地备份方案

  3. 访问控制：

    • 使用强密码
    • 启用双因素认证
    • 限制root登录

  4. Docker安全：

    • 不要使用latest标签（使用特定版本）
    • 限制容器权限
    • 定期更新镜像
