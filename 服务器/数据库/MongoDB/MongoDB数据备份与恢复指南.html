# MongoDB数据备份与恢复指南

在日常的数据库运维工作中，数据备份与恢复是非常重要的环节。MongoDB提供了多种数据备份与恢复的工具和方法，本文将详细介绍这些工具的使用方式。

# 数据备份工具

# mongodump 命令

mongodump 是 MongoDB 自带的数据备份工具，可以将数据导出为 BSON 格式。

基本命令格式：

mongodump -h IP --port 端口 -u 用户名 -p 密码 -d 数据库 -o 文件存在路径

如果想导出所有数据库，可以省略 -d 参数。

示例：

mongodump -h 127.0.0.1 --port 30216 -d test -u xxxx -p xxxxx -o home/mongodb/

# mongoexport 命令

mongoexport 可以将数据导出为 JSON 或 CSV 格式，更适合与其他系统进行数据交换。

基本命令格式：

mongoexport -h IP --port 端口 -u 用户名 -p 密码 -d 数据库 -c 表名 -f 字段

常用参数说明：

  • -f：指定导出的字段，以逗号分隔
  • -q：指定查询条件导出数据
  • --csv：指定导出为 CSV 格式

示例：

# 导出整张表
mongoexport -d test -c users -o /home/mongodb/test/users.dat

# 导出部分字段
mongoexport -d test -c users --csv -f uid,name,sex -o test/users.csv

# 根据条件导出数据
mongoexport -d test -c users -q '{uid:{$gt:1}}' -o test/users.json

# 数据恢复工具

# mongorestore 命令

mongorestore 用于恢复由 mongodump 备份的数据。

基本命令格式：

mongorestore -h IP --port 端口 -u 用户名 -p 密码 -d 数据库 --drop 文件存在路径

示例：

mongorestore -d test /home/mongodb/test

# mongoimport 命令

mongoimport 用于导入由 mongoexport 导出的数据。

基本命令格式：

# 还原整表导出的非csv文件
mongoimport -h IP --port 端口 -u 用户名 -p 密码 -d 数据库 -c 表名 --upsert --drop 文件名

# 还原部分字段的导出文件
mongoimport -h IP --port 端口 -u 用户名 -p 密码 -d 数据库 -c 表名 --upsertFields 字段 --drop 文件名

# 还原导出的csv文件
mongoimport -h IP --port 端口 -u 用户名 -p 密码 -d 数据库 -c 表名 --type 类型 --headerline --upsert --drop 文件名

示例：

# 还原导出的表数据
mongoimport -d test -c users --upsert test/users.dat

# 还原csv文件
mongoimport -d test -c users --type csv --headerline --file test/users.csv

# MongoDB 服务管理

# 启动 MongoDB 服务器

# 指定数据存储路径启动
F:\mongodb\bin\mongod --dbpath F:\mongodb\data\db

# 或者通过配置文件启动
"F:\mongodb\bin\mongod.exe" --config "F:\mongodb\mongod.cfg" --install

# 连接 MongoDB

C:\mongodb\bin\mongo.exe

# 特殊场景：YAPI 数据恢复

在使用 Docker 部署的 YAPI 环境中，可以通过以下步骤恢复数据：

# 将备份数据复制到容器中
docker cp -a /root/yapi/ mongo:/root/yapi/

# 执行数据恢复
mongorestore -u admin -p 123456 --port 27017 --authenticationDatabase admin -d yapi /root/yapi

# 总结

MongoDB 提供了完整的数据备份与恢复解决方案，通过合理使用这些工具，可以有效保障数据安全。在实际操作中，建议根据具体需求选择合适的工具和参数，定期进行数据备份，并验证恢复流程的有效性。
