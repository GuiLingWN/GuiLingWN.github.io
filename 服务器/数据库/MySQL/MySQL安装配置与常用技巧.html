# MySQL 安装配置与常用技巧

MySQL 是最流行的开源关系型数据库管理系统之一。本文将介绍在 Debian 系统上安装 MySQL 的方法，以及时区修改等常用配置技巧。

# Debian 系统安装 MySQL

在 Debian 系统上安装 MySQL，可以参考相关教程进行操作。

安装完成后，需要根据具体需求进行配置和优化。

# 时区配置

MySQL 时区设置对于确保时间数据的准确性非常重要。

# 修改 MySQL 时区

-- 设置当前会话时区
SET time_zone = '+8:00';

-- 设置全局时区
SET GLOBAL time_zone = '+08:00';

-- 刷新权限使设置生效
FLUSH PRIVILEGES;

# 查看当前时区设置

-- 查看当前会话时区
SELECT @@session.time_zone;

-- 查看全局时区
SELECT @@global.time_zone;

# 同时返回查询总数及分页数据

在分页查询中，通常需要同时获取数据和总记录数。MySQL 提供了 SQL_CALC_FOUND_ROWS 和 FOUND_ROWS() 函数来实现这一功能。

-- 查询数据并计算总行数
SELECT SQL_CALC_FOUND_ROWS * FROM sys_user LIMIT 1;

-- 获取上一次查询的总行数
SELECT FOUND_ROWS();

这种方式可以在一次查询中同时获取分页数据和总记录数，避免了分别执行两次查询。

# SQL 分组筛选前 N 条记录

在实际业务中，经常需要查询每个分组的前 N 条记录，比如每个分类的前 3 个热门商品。

# 实现思路

主要思路是：

  1. 一张表查询两次
  2. 子查询用于筛选当前记录在该组中是第几个（用序号字段区分）
  3. 主查询获取数据并排序

# 示例代码

SELECT
    title,
    exam_id,
    category,
    priority_lev
FROM
    b_exam a
WHERE
    (
        SELECT
            COUNT(*)
        FROM
            b_exam b
        WHERE
            a.category = b.category
        AND a.no <= b.no
    ) <= 3
ORDER BY
    category,
    priority_lev DESC;

# 创建函数时的常见错误解决

在 MySQL 中创建自定义函数时，可能会遇到以下错误：

This function has none of DETERMINISTIC, NO SQL, or READS SQL DATA in its definition

# 解决方法

-- 设置允许创建函数
SET GLOBAL log_bin_trust_function_creators = TRUE;

设置后即可正常创建自定义函数。

# JOIN 查询基础

SQL JOIN 是用于根据两个或多个表中的列之间的关系，从这些表中查询数据的常用方法。

常见的 JOIN 类型包括：

  • INNER JOIN：内连接，返回两个表中匹配的记录
  • LEFT JOIN：左连接，返回左表中的所有记录和右表中匹配的记录
  • RIGHT JOIN：右连接，返回右表中的所有记录和左表中匹配的记录
  • FULL OUTER JOIN：全外连接，返回两个表中的所有记录

通过合理使用 JOIN 操作，可以高效地查询关联表中的数据。

# 总结

掌握 MySQL 的安装配置和常用技巧对于数据库管理和应用开发非常重要。在实际使用中，应根据具体需求选择合适的配置和查询方法，并注意时区等细节设置，以确保数据的准确性和一致性。
