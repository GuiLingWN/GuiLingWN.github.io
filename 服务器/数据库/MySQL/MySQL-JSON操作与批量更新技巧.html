# MySQL JSON 操作与批量更新技巧

在现代 Web 应用开发中，MySQL 的 JSON 功能和批量操作技巧对于提高开发效率和数据处理能力非常重要。本文将介绍一些实用的 MySQL JSON 操作和批量更新技巧。

# JSON 数据查找与操作

MySQL 5.7 及以上版本支持 JSON 数据类型和相关函数，可以方便地操作 JSON 数据。

# JSON 查找操作

-- JSON_CONTAINS 函数用于检查 JSON 文档是否包含特定值
SELECT *, JSON_CONTAINS(allocation_rule, '{"name": "测试"}', "$") FROM b_exhibition;

-- 查找 JSON 数组中包含特定值的元素
SELECT JSON_CONTAINS('[{"name": "博物馆", "weight": 4}, {"name": "Rokid", "weight": 3}, {"name": "宽阅科技", "weight": 3}]', '{"weight": 3}', "$");

-- 使用 json_search 查找 JSON 中的字符串值
SELECT json_search('[{"name": "博物馆", "weight": 4}, {"name": "Rokid", "weight": 3}, {"name": "宽阅科技", "weight": 3}]', 'one', "Rokid", NULL, '$[*].name');

需要注意的是，json_search 函数仅可用于查找字符串值。

# 批量更新操作

在处理大量数据时，使用批量更新可以显著提高效率。

# CASE WHEN 语句实现批量更新

UPDATE `user`
SET age = CASE id
    WHEN 1 THEN '22'
    WHEN 2 THEN '22'
    WHEN 3 THEN '22'
    WHEN 4 THEN '22'
END 
WHERE id IN(1,2,3,4);

通用语法格式：

UPDATE (表名)
SET (更新字段) = CASE (被更新字段)
    WHEN (被更新字段值) THEN (更新字段值)
    ...
END 
WHERE (被更新字段) IN ((被更新字段值)...);

这种方式比逐条更新数据效率更高，特别是在需要更新大量记录时。

# 字符串处理与批量修改

在处理图片链接或其他字符串数据时，经常需要进行批量替换操作。

# 图片参数批量过滤修改

-- 从图片链接中移除参数并添加新的处理参数
SELECT pro_cover, CONCAT(
    TRIM(TRAILING SUBSTR(pro_cover FROM INSTR(pro_cover,'?')) FROM pro_cover),
    '?x-oss-process=style/300_300'
) FROM b_product LIMIT 10;

-- 执行更新操作
UPDATE b_product 
SET pro_cover = CONCAT(
    TRIM(TRAILING SUBSTR(pro_cover FROM INSTR(pro_cover,'?')) FROM pro_cover),
    '?x-oss-process=style/300_300'
);

# 字符串替换操作

-- 查询替换结果
SELECT cover, REPLACE(cover, 'https://file2.wutiankeji.com/', 'https://file.wxfxz.cn/mini-program/') 
FROM b_exam;

-- 执行替换更新
UPDATE b_exam 
SET advertise = REPLACE(advertise, 'https://file2.wutiankeji.com/', 'https://file.wxfxz.cn/mini-program/');

# 查询列表中不存在数据库中的数据

在数据校验场景中，经常需要找出列表中不存在于数据库中的数据。

-- 创建临时表存储待校验数据
DROP TABLE IF EXISTS account_tmp;
CREATE TEMPORARY TABLE account_tmp (`account` varchar(128));

-- 插入待校验数据
INSERT INTO account_tmp(`account`) VALUES('QM-1'),('QM-3'),('QM-4'),('QM-');

-- 查询不存在于数据库中的数据
SELECT account_tmp.account 
FROM account_tmp 
LEFT JOIN sys_business sb ON sb.account=account_tmp.account AND sb.is_active=1 
WHERE sb.account IS NULL;

-- 清理临时表
DROP TABLE IF EXISTS account_tmp;

如果后续需要处理存在的数据，则需要全部查出再遍历。但仅仅为了校验是否存在，使用这种方式效率更高。

# 总结

通过合理使用 MySQL 的 JSON 操作函数、批量更新技巧和字符串处理功能，可以大大提高数据处理的效率。在实际应用中，应根据具体需求选择合适的方法，并注意 SQL 语句的性能优化。
