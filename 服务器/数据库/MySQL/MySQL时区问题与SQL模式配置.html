# MySQL 时区问题与 SQL 模式配置

在使用 MySQL 数据库时，时区设置和 SQL 模式配置是两个常见但重要的问题。本文将详细介绍如何解决 MySQL 时区问题以及配置 SQL 模式。

# MySQL 时区问题

时区问题可能导致应用程序与数据库之间的时间数据不一致，影响业务逻辑的正确性。

# 查看当前时区设置

-- 查看当前时间
SELECT NOW();

-- 查看时区相关变量
SHOW VARIABLES LIKE '%time_zone%';

# 解决方法

# 步骤一：修改 Java 应用中的时区

在 JDBC 连接字符串中设置时区：

// serverTimezone 可以设置为北京时间 GMT%2B8、上海时间 Asia/Shanghai 或者香港时间 Hongkong
url: jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf8&useSSL=true

# 步骤二：修改 MySQL 数据库的时区

方法一：使用命令（临时设置）

-- 设置当前会话时区
SET time_zone = '+8:00';

-- 设置全局时区
SET GLOBAL time_zone = '+8:00';

-- 刷新权限
FLUSH PRIVILEGES;

优点：不需要重启 MySQL 服务 缺点：一旦 MySQL 服务重启，设置就会消失

方法二：修改配置文件（永久设置） 在 MySQL 配置文件中添加：

[mysqld]
# 设置默认时区
default-time_zone='+8:00'

优点：永久保存设置 缺点：需要重启 MySQL 服务才能生效

# Docker 环境下的时区配置

在 Docker 环境中，可以通过以下方式修改时区配置：

# 复制本地配置文件到容器中，覆盖容器内部的 my.cnf
docker cp /root/my.cnf container_id:/etc/mysql/

# SQL 模式配置

MySQL 的 SQL 模式（sql_mode）定义了 MySQL 应该支持的 SQL 语法和应该执行的数据验证检查。

# ONLY_FULL_GROUP_BY 问题

在使用 GROUP BY 语句时，可能会遇到以下错误：

this is incompatible with sql_mode=only_full_group_by

# 解决方案

-- 查看当前全局 SQL 模式
SELECT @@GLOBAL.sql_mode;

-- 设置新的 SQL 模式（移除 ONLY_FULL_GROUP_BY）
SET @@global.sql_mode = 'STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION';

# 常见的 SQL 模式选项

  • STRICT_TRANS_TABLES：对事务性存储引擎启用严格模式
  • NO_ZERO_IN_DATE：不允许日期中的零值
  • NO_ZERO_DATE：不允许 '0000-00-00' 日期
  • ERROR_FOR_DIVISION_BY_ZERO：除零时产生错误而不是警告
  • NO_AUTO_CREATE_USER：禁止 GRANT 自动创建用户
  • NO_ENGINE_SUBSTITUTION：当指定的存储引擎不可用时产生错误

# 字符集排序规则问题

在处理不同字符集的数据时，可能会遇到排序规则不匹配的问题。

# 错误信息

Illegal mix of collations (utf8mb4_general_ci,IMPLICIT) and (utf8mb4_0900_ai_ci,IMPLICIT) for operation '='

# 解决方法

在查询中显式指定排序规则：

SELECT * FROM table1 t1 
JOIN table2 t2 ON t1.column COLLATE utf8mb4_general_ci = t2.column;

  1. 将文件中的所有 utf8mb4_0900_ai_ci 替换为 utf8mb4_general_ci
  2. 将 utf8mb4 替换为 utf8

# 总结

正确配置 MySQL 的时区和 SQL 模式对于确保数据的一致性和应用的稳定性非常重要。在实际使用中，应根据具体需求选择合适的配置方式，并注意以下几点：

  1. 合理设置时区，确保应用与数据库时间一致
  2. 根据业务需求配置 SQL 模式
  3. 注意字符集和排序规则的兼容性
  4. 在 Docker 环境中正确处理配置文件
  5. 定期检查和优化数据库配置

通过合理配置这些参数，可以避免很多常见的数据库问题，提高系统的稳定性和可靠性。
