# Nginx 转发 SSH 与 MySQL 配置指南

Nginx 不仅可以作为 HTTP 服务器和反向代理，还可以通过 stream 模块实现 TCP/UDP 流量的转发，这使得 Nginx 能够转发 SSH、MySQL 等非 HTTP 协议的流量。

# Stream 模块配置

要使用 Nginx 转发 SSH 和 MySQL，需要确保 Nginx 编译时包含了 stream 模块。

# 转发 SSH 连接

events {    
    worker_connections  1024;
    accept_mutex on;
}

stream { 
    upstream ssh { 
        server IP:22;
    }
    
    server { 
        listen 9898;
        proxy_pass ssh;
        proxy_connect_timeout 5s;
        proxy_timeout 30s;
    }
}

# 转发 MySQL 连接

stream {
    server {
        listen 3306;
        proxy_connect_timeout 5s;
        proxy_timeout 300s;
        proxy_pass IP:3306;
    }
}

# 检查和启用 Stream 模块

# 检查 Nginx 是否支持 Stream 模块

nginx -V 2>&1 | grep -- --with-stream

如果输出包含 --with-stream，说明 Nginx 支持 stream 模块。

如果遇到以下错误：

nginx: [emerg] unknown directive "stream" in /etc/nginx/nginx.conf:224

说明 Nginx 是以动态模块的方式支持 stream 模块：

--with-stream=dynamic --with-stream_ssl_module

在这种情况下，需要在 /etc/nginx/nginx.conf 文件的最顶端添加：

load_module modules/ngx_stream_module.so;

# Ubuntu 系统下 Nginx 安装与卸载

# 安装 Nginx

# 安装
apt install nginx

# 查看安装版本
nginx -v

# 启动 Nginx
service nginx start

Nginx 安装完成后的文件位置：

  • /usr/sbin/nginx：主程序
  • /etc/nginx：存放配置文件
  • /usr/share/nginx：存放静态文件
  • /var/log/nginx：存放日志

# 卸载 Nginx

# 删除 nginx，--purge 包括配置文件
sudo apt-get --purge remove nginx

# 自动移除全部不使用的软件包
sudo apt-get autoremove

# 罗列出与 nginx 相关的软件
dpkg --get-selections|grep nginx

# 删除查询出与 nginx 有关的软件
sudo apt-get --purge remove nginx
sudo apt-get --purge remove nginx-common
sudo apt-get --purge remove nginx-full

# 使用场景

通过 Nginx 转发 SSH 和 MySQL 连接，可以实现以下场景：

  1. 安全访问：通过 Nginx 代理访问内网的 SSH 和 MySQL 服务，避免直接暴露服务端口
  2. 负载均衡：对多个 MySQL 实例进行负载均衡
  3. 访问控制：通过 Nginx 的访问控制功能限制对 SSH 和 MySQL 的访问
  4. 日志记录：记录 SSH 和 MySQL 的访问日志，便于审计和监控

# 注意事项

  1. 确保 Nginx 版本支持 stream 模块（如 1.18.0 及以上版本）
  2. 配置完成后需要重启 Nginx 服务使配置生效
  3. 注意防火墙设置，确保转发端口可以正常访问
  4. 合理设置超时时间，避免连接异常断开

通过以上配置，可以实现通过 Nginx 转发 SSH 和 MySQL 连接，为内网服务提供安全的外部访问方式。
