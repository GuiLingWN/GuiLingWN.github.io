# Nginx 日志分析与处理技巧

Nginx 日志是系统运维和问题排查的重要工具。通过分析访问日志和错误日志，我们可以了解系统的运行状况、用户行为以及潜在问题。本文将介绍一些实用的 Nginx 日志分析和处理技巧。

# 日志分析基础

Nginx 访问日志记录了每个请求的详细信息，通过分析这些信息可以了解接口访问情况、用户行为等。

# 使用 awk 统计接口访问次数

# 统计接口访问次数并排序
awk '{split($7, a, "?"); print a[1]}' access.log | sort | uniq -c | sort -nr

# 匹配指定接口
awk '$7 ~ /^\/saas_test\// {split($7, a, "?"); print a[1]}' access.log | sort | uniq -c | sort -nr

# 限制时间区间
awk '$7 ~ /^\/saas_test\// && $4 >= "[12/Jul/2024:15:22:00" && $4 <= "[12/Jul/2024:15:24:00" {split($7, a, "?"); print a[1]}' access.log | sort | uniq -c | sort -nr

# 多时间段日志分析

# 保存3个时间段统计到文件
awk '$7 ~ /^\/saas_test\// && $4 >= "[15/Jul/2024:13:05:00" && $4 <= "[15/Jul/2024:13:15:00" {split($7, a, "?"); print a[1]}' access.log | sort | uniq -c > period1.log
awk '$7 ~ /^\/saas_test\// && $4 >= "[15/Jul/2024:13:50:00" && $4 <= "[15/Jul/2024:13:59:00" {split($7, a, "?"); print a[1]}' access.log | sort | uniq -c > period2.log
awk '$7 ~ /^\/saas_test\// && $4 >= "[15/Jul/2024:14:50:00" && $4 <= "[15/Jul/2024:14:59:00" {split($7, a, "?"); print a[1]}' access.log | sort | uniq -c > period3.log

# 提取 URL 并排序
awk '{print $2}' period1.log | sort > period1_urls.log
awk '{print $2}' period2.log | sort > period2_urls.log
awk '{print $2}' period3.log | sort > period3_urls.log

# 找出共同的 URL
comm -12 period1_urls.log period2_urls.log period3_urls.log > common_urls.log

# 合并多个日志文件分析

# 合并多个文件
cat access1.log access2.log access3.log > access.log

# 筛选 nginx 接口到相同文件
awk '$4 >= "[15/Jul/2024:13:05:00" && $4 <= "[15/Jul/2024:13:15:00" && $7 ~ /\/saas_test\// {split($7, a, "?"); print a[1]}' access.log | sort | uniq >> periodAll.log
awk '$4 >= "[15/Jul/2024:13:50:00" && $4 <= "[15/Jul/2024:13:59:00" && $7 ~ /\/saas_test\// {split($7, a, "?"); print a[1]}' access.log | sort | uniq >> periodAll.log
awk '$4 >= "[15/Jul/2024:14:50:00" && $4 <= "[15/Jul/2024:14:59:00" && $7 ~ /\/saas_test\// {split($7, a, "?"); print a[1]}' access.log | sort | uniq >> periodAll.log

# 统计相同行排序，出现3次即为全部都有
awk '{count[$0]++} END {for (line in count) print count[line], line}' periodAll.log | sort -nr

# 日志轮转

为了防止日志文件过大，建议配置日志轮转。Nginx 支持通过 logrotate 工具进行日志轮转管理。

更多关于日志轮转的详细信息，可以参考相关文档。

# 总结

通过合理分析 Nginx 日志，我们可以：

  1. 了解接口访问情况，发现热点接口
  2. 分析用户行为，优化用户体验
  3. 发现潜在问题，及时处理异常
  4. 进行容量规划，合理分配资源

掌握这些日志分析技巧，可以大大提高系统运维效率和问题排查速度。
