# Nginx 反向代理路径处理与子域名配置

在使用 Nginx 作为反向代理服务器时，路径处理和子域名配置是两个常见的需求。本文将详细介绍如何正确配置这些功能。

# 反向代理路径处理

在配置 Nginx 反向代理时，路径的处理方式对请求的转发结果有重要影响。

# 路径匹配与转发

location ~ ^/play(/?)(.*) {
    proxy_pass http://127.0.0.1:9000/$2;
}

这个配置的含义是：将 /play/xx 的请求转发到 http://127.0.0.1:9000/xx。

# 更简洁的配置方式

location /play {
    proxy_pass http://127.0.0.1:9000/;
}

注意结尾的 / 符号的重要性。

# 路径处理规则

当 proxy_pass 指令后面加上 / 时，表示使用绝对根路径，Nginx 不会将 location 中匹配的路径部分传递给后端：

# 不传递匹配路径
proxy_pass http://backend/;

# 传递匹配路径
proxy_pass http://backend;

例如：

  • 请求 /play/test，使用 proxy_pass http://127.0.0.1:9000/; 会转发到 http://127.0.0.1:9000/test
  • 请求 /play/test，使用 proxy_pass http://127.0.0.1:9000; 会转发到 http://127.0.0.1:9000/play/test

# 子域名访问对应目录配置

通过 Nginx 的变量功能，可以实现根据子域名访问对应目录的功能。

# 方法一：使用正则表达式提取子域名

server {
    listen 80;
    server_name *.saas.shhuih.cn;
    
    # 使用正则表达式提取子域名
    if ($host ~* (.*)\.(.*)\.(.*)\.(.*)) {
        set $subdomain $1;
    }
    
    location / {
        root html/web/$subdomain;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}

# 方法二：直接使用主机名变量

server {
    listen 80;
    server_name *.saas.shhuih.cn;
    
    location / {
        root html/web/$host;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}

# 日志处理定时任务脚本

为了防止日志文件过大，可以设置定时任务定期处理和轮转日志文件。

Windows 环境下的批处理脚本示例：

@echo off
cd C:\nginx
set today=%date:~2,2%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%
set path=C:\nginx\logs\access\%today%
move C:\nginx\logs\access.log %path%.log
nginx -s reload

该脚本的作用：

  1. 切换到 Nginx 目录
  2. 生成当前日期时间戳
  3. 创建日志备份路径
  4. 移动当前访问日志到备份位置
  5. 重新加载 Nginx 配置，生成新的日志文件

Nginx 安装后的文件位置：

  • /usr/sbin/nginx：主程序
  • /etc/nginx：存放配置文件
  • /usr/share/nginx：存放静态文件
  • /var/log/nginx：存放日志

# 总结

通过合理配置 Nginx 的反向代理路径处理和子域名访问，可以实现灵活的请求转发和资源访问。在实际应用中，需要注意：

  1. 正确理解 proxy_pass 指令中斜杠的作用
  2. 合理使用 Nginx 变量提取和处理请求信息
  3. 定期处理日志文件，防止磁盘空间不足
  4. 根据实际需求选择合适的配置方式

这些配置技巧可以帮助我们更好地利用 Nginx 的强大功能，构建高效稳定的 Web 服务。
