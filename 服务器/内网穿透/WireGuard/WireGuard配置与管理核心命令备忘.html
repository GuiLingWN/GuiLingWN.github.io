# WireGuard 配置与管理核心命令备忘

WireGuard 以其简洁、高效和安全而备受青睐，已成为许多人替代传统 VPN 的首选。在配置和日常管理 WireGuard 的过程中，会涉及到一系列的命令，从生成密钥到管理服务，再到配置防火墙规则。为了方便查阅，我将这些核心操作整理成了一份备忘录。

# 一、密钥生成

WireGuard 的安全基石在于其加密密钥对。每个参与通信的对等端（Peer）都需要有一对自己的公钥和私钥。

  • 生成单个密钥对

    这是一个非常实用的“一条龙”命令，它会生成一个私钥，并立即从中派生出对应的公钥。

    # 生成私钥并保存到 private.key，同时生成公钥并保存到 public.key
wg genkey | tee private.key | wg pubkey > public.key

  • 批量生成密钥

    如果你需要为多个客户端设备生成密钥，可以像下面这样操作：

    # 1. 生成服务器的密钥对
wg genkey > server.key
wg pubkey < server.key > server.key.pub

# 2. 为 iPhone 生成密钥对
wg genkey > iphone.key
wg pubkey < iphone.key > iphone.key.pub

# 3. 为 MacBook 生成密钥对
wg genkey > macos.key
wg pubkey < macos.key > macos.key.pub

  • 生成预共享密钥 (Preshared Key)

    为了增加一层额外的安全保障（抵御量子计算攻击），可以在对等端之间配置一个预共享密钥。

    wg genpsk > preshared.key

    这个生成的密钥需要在服务器和客户端的 [Peer] 配置块中同时指定。

# 二、服务管理

管理 WireGuard 接口通常有两种方式：使用 wg-quick 脚本或 systemd 服务。

# 使用 wg-quick

wg-quick 是一个便捷的脚本，它可以根据 /etc/wireguard/ 目录下的配置文件（如 wg0.conf）来快速启动或关闭一个 WireGuard 接口。

# 启动名为 wg0 的接口
wg-quick up wg0

# 关闭名为 wg0 的接口
wg-quick down wg0

# 查看当前所有 WireGuard 接口的状态（包括连接信息）
wg show
# 或者直接用 wg 命令
wg

# 使用 systemd

为了让 WireGuard 能开机自启，最好将其作为 systemd 服务来管理。

wg-quick 工具已经为我们准备好了 systemd 的服务单元模板 wg-quick@.service。

# 启动 wg0 接口
sudo systemctl start wg-quick@wg0

# 停止 wg0 接口
sudo systemctl stop wg-quick@wg0

# 重启 wg0 接口
sudo systemctl restart wg-quick@wg0

# 开机自启 wg0 接口
sudo systemctl enable wg-quick@wg0

# 查看服务状态
sudo systemctl status wg-quick@wg0

# 三、防火墙与路由 (iptables)

要让 WireGuard 客户端能够通过服务器访问外部网络，关键在于配置正确的 iptables 规则来实现网络地址转换（NAT）和数据包转发。

假设：

  • WireGuard 接口名为 wg0。
  • 服务器的公网网卡接口名为 ens18。
  • WireGuard 的子网是 10.0.8.0/24。
# 1. 允许 FORWARD 链接受来自和去往 wg0 接口的数据包
iptables -A FORWARD -i wg0 -j ACCEPT
iptables -A FORWARD -o wg0 -j ACCEPT

# 2. 在 POSTROUTING 链上，将源地址为 wg0 子网的数据包进行 SNAT (源地址转换)，
#    伪装成从公网网卡 ens18 出去的样子
iptables -t nat -A POSTROUTING -o ens18 -j MASQUERADE

重要提示：这些规则在系统重启后会丢失。你需要使用 iptables-persistent 之类的工具来持久化这些规则。

同时，别忘了在 INPUT 链上放行 WireGuard 的监听端口（默认为 UDP 51820）：

iptables -A INPUT -p udp --dport 51820 -j ACCEPT
iptables-save

# 四、使用 Docker 和 wg-easy (带 Web UI)

对于不喜欢直接在宿主机上配置和管理的用户，wg-easy 是一个非常棒的 Docker 项目。它提供了一个简单的 Web UI 来管理客户端，大大降低了使用门槛。

docker run -d \
  --name=wg-easy \
  -e WG_HOST=你的服务器公网IP \
  -e PASSWORD=你的UI登录密码 \
  -v /path/to/your/config:/etc/wireguard \
  -p 51820:51820/udp \
  -p 51821:51821/tcp \
  --cap-add=NET_ADMIN \
  --cap-add=SYS_MODULE \
  --sysctl="net.ipv4.conf.all.src_valid_mark=1" \
  --sysctl="net.ipv4.ip_forward=1" \
  --restart unless-stopped \
  weejewel/wg-easy
  • WG_HOST: 必须设置为你的服务器的公网 IP 或域名。
  • PASSWORD: 用于登录 Web UI 的密码。
  • -v /path/to/your/config:/etc/wireguard: 将 WireGuard 的配置持久化到宿主机，防止容器销毁后配置丢失。

启动后，访问 http://你的服务器IP:51821 即可进入管理界面。
