# FRP内网穿透配置完全指南

FRP（Fast Reverse Proxy）是一款高性能的反向代理应用，可以帮助我们实现内网穿透，将内网服务暴露到公网。本文将详细介绍FRP的配置与使用方法。

# FRP基本概念

FRP主要由两个组件构成：

  • frps：服务端程序，运行在具有公网IP的服务器上
  • frpc：客户端程序，运行在内网环境中

# 服务端配置

FRP服务端的核心配置文件是frps.ini，一个基础配置示例如下：

[common]
bind_port = 12450
vhost_http_port = 8081

配置说明：

  • bind_port：FRP服务端监听的端口，用于与客户端通信
  • vhost_http_port：HTTP访问的端口

服务端启动命令：

nohup ./frps -c frps.ini >/dev/null 2>&1 &

# 客户端配置

客户端配置文件frpc.ini包含了具体的穿透规则，以下是一个配置示例：

[common]
server_addr = your_server_ip
server_port = 12450

token = your_token

[yf_jenkins]
type = http
local_port = 8081
custom_domains = jenkins.your_domain

[proxy2.9]
type = http
local_ip = 192.168.2.9
local_port = 8080
custom_domains = 9.your_domain

配置说明：

  • [common]部分配置服务端连接信息
  • server_addr：FRP服务端的公网IP地址
  • server_port：FRP服务端监听的端口
  • token：用于验证的令牌
  • 其他部分为具体的穿透规则，如[yf_jenkins]为一个HTTP穿透规则

# 客户端开机自启动配置

为了让FRP客户端在系统启动时自动运行，我们可以配置systemd服务：

创建服务文件：

vim /usr/lib/systemd/system/frpc.service

服务配置内容：

[Unit]
Description=Frp Client Service
After=network.target

[Service]
Type=simple
User=nobody
Restart=on-failure
RestartSec=5s
ExecStart=/root/frp/frpc -c /root/frp/frpc.ini
ExecReload=/root/frp/frpc reload -c /root/frp/frpc.ini

[Install]
WantedBy=multi-user.target

启用服务：

systemctl daemon-reload
systemctl enable frpc
systemctl start frpc

# Nginx反向代理配置

为了更好地管理和访问FRP穿透的服务，我们通常会配合Nginx使用：

server {
    listen       80;
    server_name *.frp.your_domain;
    location / {
        proxy_pass  http://127.0.0.1:8081/;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Host $http_host;
    }
}

# FRP转发WireGuard流量

FRP也支持UDP协议的穿透，可以用于转发WireGuard等基于UDP的应用：

服务端配置：

[common]
bind_addr = 0.0.0.0
bind_port = 12580
bind_udp_port  = 12581
vhost_http_port = 8081

客户端配置：

[common]
server_addr = your_server_ip
server_port = 12580

[yf_wg_udp]
type = udp
local_ip = 127.0.0.1
local_port = 12588
remote_port = 12588
custom_domains = wg.your_domain

# 总结

FRP是一个功能强大的内网穿透工具，通过合理的配置可以实现HTTP、TCP、UDP等多种协议的穿透。在实际使用中，需要注意安全配置，如设置强密码、限制访问IP等，以确保服务的安全性。
