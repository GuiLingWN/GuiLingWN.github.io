# NestJS请求链路追踪中间件实现

在微服务架构或复杂的后端应用中，请求链路追踪是一项重要的技术。通过为每个请求分配唯一的 ID，可以在日志中快速定位和追踪问题。这里分享一个基于 Node.js AsyncLocalStorage 的 NestJS 请求链路追踪中间件实现。

# 安装依赖

首先安装请求 ID 生成库：

npm i express-request-id --save

# 中间件实现

创建 tracing.middleware.ts 文件：

import { Injectable, NestMiddleware } from '@nestjs/common'
import { AsyncLocalStorage } from 'async_hooks'

/**
 * 请求链路追踪中间件
 */
@Injectable()
export class TracingMiddleware implements NestMiddleware {
  private storage: AsyncLocalStorageService
  constructor() {
    this.storage = AsyncLocalStorageService.getInstance()
    let log = console.log
    let error = console.error
    console.log = (...args) => {
      let rid = this.storage.get('requestId')
      rid && args.unshift(`[${rid}]`)
      log(...args)
    }
    console.error = (...args) => {
      let rid = this.storage.get('requestId')
      rid && args.unshift(`[${rid}]`)
      error(...args)
    }
  }
  async use(req: any, res: any, next: () => void) {
    this.storage.enable(() => {
      this.storage.set('requestId', req.id)
      next()
    })
  }
}

/**
 * AsyncLocalStorageService链路存储单例
 */
export class AsyncLocalStorageService {
  private static instance: AsyncLocalStorageService
  private asyncLocalStorage: AsyncLocalStorage<Object>
  constructor() {
    this.asyncLocalStorage = new AsyncLocalStorage()
  }
  public static getInstance() {
    if (!this.instance) {
      this.instance = new AsyncLocalStorageService()
    }
    return this.instance
  }
  public enable(callback) {
    this.asyncLocalStorage.run({}, callback)
  }
  public get(key) {
    return (this.asyncLocalStorage.getStore() && this.asyncLocalStorage.getStore()[key]) || ''
  }
  public set(key, value) {
    this.asyncLocalStorage.getStore()[key] = value
  }
}

# 注册中间件

在 app.module.ts 中注册中间件：

export class AppModule {
  public configure(consumer: MiddlewareConsumer) {
    consumer.apply(TracingMiddleware).forRoutes('*')
  }
}

# 启用请求 ID 生成

在 main.ts 中启用请求 ID 生成：

import * as addRequestId from 'express-request-id'

app.use(addRequestId())

# 工作原理

  1. 请求 ID 生成：express-request-id 为每个请求生成唯一 ID 并附加到 req.id
  2. 上下文存储：使用 AsyncLocalStorage 在异步调用链中保持请求上下文
  3. 日志增强：重写 console.log 和 console.error，自动在日志前添加请求 ID
  4. 链路追踪：同一请求的所有日志都会带有相同的请求 ID，方便追踪

# 使用效果

启用后，所有日志输出会自动带上请求 ID：

[abc123] 用户登录请求
[abc123] 查询用户信息
[abc123] 登录成功
[xyz456] 获取订单列表
[xyz456] 查询数据库

这样就能清晰地看到每个请求的完整调用链路。

# 小结

这个中间件实现了基础的请求链路追踪功能，适用于大多数场景。在生产环境中，你可能还需要考虑：

  • 与分布式追踪系统（如 Jaeger、Zipkin）集成
  • 性能优化和内存管理
  • 更完善的日志格式化
  • 支持自定义日志库（如 Winston、Pino）
