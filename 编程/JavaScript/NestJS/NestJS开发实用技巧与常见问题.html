# NestJS开发实用技巧与常见问题

在使用NestJS进行Node.js后端开发时，遇到了一些实用技巧和常见问题。这里整理了调试方法、模块管理、TypeORM使用等方面的经验，供参考。

# 调试技巧：使用Chrome DevTools调试NestJS

# 方法一：手动配置Chrome DevTools

NestJS支持使用Chrome DevTools进行调试。首先在启动时添加debug参数，然后访问chrome://inspect页面，即可连接到Node进程进行调试。

具体步骤：

  1. 以debug模式启动NestJS应用
  2. 打开Chrome浏览器，访问 chrome://inspect
  3. 在Remote Target中找到你的应用并点击inspect

这样就可以使用Chrome强大的调试功能来检查代码、设置断点、查看变量等。

# 方法二：使用Chrome扩展

更便捷的方式是安装Chrome扩展NIM (Node Inspector Manager)，它可以自动检测并连接到Node调试进程。

参考资料：

  • Node.js官方调试文档 (opens new window)

# TypeORM使用技巧

# FIND_IN_SET查询

在处理MySQL的SET类型字段时，可以使用TypeORM的Raw函数实现FIND_IN_SET查询：

使用Repository的find方法：

gradeLevels: Raw(col => `FIND_IN_SET('${gradeLevel}', ${col}) > 0`)

使用QueryBuilder：

.andWhere('FIND_IN_SET(:gradeLevel, question.gradeLevels) > 0', { gradeLevel })

实际应用示例：

async orderByDeviceNo({ deviceNo, orderStatus }) {
  let list = await OrderEntity.find({
    status: orderStatus,
    deviceNo: Raw((col) => `FIND_IN_SET('${deviceNo}', ${col}) > 0`)
  });
  return { list };
}

# 防SQL注入：使用转义函数

当需要在原生SQL中使用用户输入的参数时，可以使用TypeORM提供的escape函数进行转义：

import { getConnection } from 'typeorm';
const escapedValue = getConnection().driver.escape('前端参数');

# 统计函数查询

使用QueryBuilder进行分组统计查询：

let minProStockInfo = await getRepository(BusinessOnSaleEntity)
  .createQueryBuilder()
  .select('price')
  .addSelect('SUM(1)', 'stock')
  .where({ isActive: true, sku, status: ON_SALE_STATUS.ON_SALE })
  .groupBy('price')
  .orderBy('price', 'ASC')
  .getRawOne();

返回结果示例：

{
  "minProStockInfo": {
    "price": "999",
    "stock": "9"
  }
}

# 常见问题解决

# 循环依赖问题

当模块之间存在相互引用时，需要使用forwardRef来解决循环依赖：

// 在module和service中都配置forwardRef
@Module({
  imports: [forwardRef(() => OtherModule)]
})

# DTO验证不生效

使用class-validator进行DTO验证时，如果验证不生效，检查以下几点：

  1. 确保路由控制器中已正确指定DTO类型
  2. 确保全局ValidationPipe已配置
  3. 检查DTO类中的装饰器是否正确引入

# 编译时自动拷贝静态资源

有时需要将证书、配置文件等静态资源拷贝到编译输出目录，可以在nest-cli.json中配置：

{
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "assets": ["utils/apiclient_cert.p12"]
  }
}
