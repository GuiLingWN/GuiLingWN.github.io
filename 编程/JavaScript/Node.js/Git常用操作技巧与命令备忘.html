# Git常用操作技巧与命令备忘

在日常开发中，Git 的一些不太常用但很实用的命令和技巧，往往容易被忘记。这里整理了一些我在实践中积累的 Git 操作笔记，供需要时快速查阅。

# 合并本地分支

将某个特性分支合并到当前分支：

git merge feature-branch

# 忽略提交已受版本控制的文件

有时候需要临时忽略某些已经被 Git 追踪的文件，但又不想删除它们，可以使用以下几种方法：

# 假设未修改（推荐）

# 标记文件为"假设未修改"
git update-index --assume-unchanged <file>

# 取消假设未修改
git update-index --no-assume-unchanged <file>

# 查询所有"假设未修改"的文件
# Linux/Mac:
git ls-files -v | grep '^h'
# Windows:
git ls-files -v | findstr "^h"

# 停止追踪文件

# Git 会停止追踪该文件，下次提交时会显示为"被删除"
git rm --cached <file>

# 使用 stash（临时隐藏更改）

git stash push <file>
git stash pop

更多讨论可参考：ChatGPT 对话 (opens new window)

# 本地 Git 目录克隆

在本地磁盘之间克隆仓库：

git clone d:/SourceRepository d:/DestinationRepository

说明：

  • d:/SourceRepository 为源仓库路径
  • d:/DestinationRepository 为目标路径（可选参数，默认为当前目录）
  • 注意：目标目录不能已经存在，否则克隆会失败

# 远程仓库强制覆盖本地

当本地代码出现问题，需要用远程分支完全覆盖本地时：

git fetch --all
git reset --hard origin/master  # master 改为对应的分支名
git pull

警告： 此操作会丢失所有本地未提交的更改，请谨慎使用！

# 清空 Git 缓存

当修改了 .gitignore 文件后，需要清除缓存使其生效：

git rm -r --cached .
git add .

# 使用 BFG 清除已上传的敏感信息

如果不小心将密码、密钥等敏感信息提交到了 GitHub，可以使用 BFG 工具清除历史记录。

# 创建替换规则文件

创建 passwords.txt 文件，定义替换规则：

PASSWORD1                       # 替换为 '***REMOVED***'（默认）
PASSWORD2==>examplePass         # 替换为 'examplePass'
PASSWORD3==>                    # 替换为空字符串
regex:password=\w+==>password=  # 使用正则表达式替换
regex:\r(\n)==>$1               # 将 Windows 换行符替换为 Unix 换行符

# 执行清理

# 镜像克隆仓库
git clone --mirror https://github.com/用户名/仓库名.git

# 使用 BFG 工具替换敏感信息
java -jar bfg.jar --replace-text passwords.txt --no-blob-protection 仓库名.git

# 进入仓库目录并推送
cd 仓库名.git
git push

# 小结

这些命令和技巧在特定场景下非常实用。建议根据实际需求选择合适的方法，特别是涉及强制操作和敏感信息清理时，一定要谨慎操作并做好备份。
