# JD脚本相关配置与安装

在使用京东自动化脚本时，经常需要用到一些配置信息和安装命令。这里整理了常用的 User Agent 和快速安装脚本。

# User Agent 配置

在进行 JD 接口请求时，可能需要使用特定的 User Agent 字符串：

jd-user-agent: okhttp/3.12.1;jdmall;android;version/9.5.2;build/87971;screen/1080x2276;os/11;network/wifi;

这个 UA 字符串模拟的是京东 Android 客户端的请求。

# Linux 系统安装

在 Linux 服务器上快速安装 JD 脚本环境：

wget -q https://gitee.com/highdimen/js_tool/raw/A1/resource/install_scripts/linux_install_jd.sh -O linux_install_jd.sh && chmod +x linux_install_jd.sh && bash linux_install_jd.sh

这个命令会自动下载安装脚本并执行，完成环境配置。

# Docker 安装

如果你更喜欢使用 Docker 容器化部署：

wget -q https://gitee.com/highdimen/js_tool/raw/A1/resource/install_scripts/docker_install_jd.sh -O docker_install_jd.sh && chmod +x docker_install_jd.sh && bash docker_install_jd.sh

Docker 方式部署更加灵活，便于管理和迁移。

# 注意事项

  • 使用这些脚本前，请确保理解其功能和潜在风险
  • 建议先在测试环境运行，确认无误后再部署到生产环境
  • 脚本来源于第三方，使用时请注意安全性

# 小结

这些命令和配置可以帮助你快速搭建 JD 脚本运行环境。根据实际需求选择 Linux 直接安装或 Docker 容器化部署方式。
