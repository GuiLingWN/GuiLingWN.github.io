# Node.js捕获未处理的Promise异常

在 Node.js 应用中，未捕获的 Promise 异常是一个常见但棘手的问题。这类异常往往没有完整的错误堆栈信息，给调试带来很大困扰。本文介绍如何正确捕获和处理这些异常。

# 问题描述

当出现 UnhandledPromiseRejectionWarning: Unhandled promise rejection 错误时，往往缺少详细的堆栈信息，难以定位问题源头。

实际上，如果正确捕获，error 对象中是包含完整的 stack 堆栈信息的。

# 解决方案

# 方法一：自定义错误堆栈处理

通过重写 Error.prepareStackTrace，可以在错误发生时获取完整堆栈：

Error.prepareStackTrace = function (error) {
  console.log(error.message, error.stack)
  return error
}

# 方法二：全局异常监听（推荐）

在应用启动时添加全局异常监听器，捕获所有未处理的异常：

// 捕获未捕获的同步异常
process.on('uncaughtException', (error) => {
  console.error('未处理异常:', error)
  // 这里可以添加自定义的异常处理逻辑
  // 例如：记录日志、发送告警、优雅退出等
})

// 捕获未处理的 Promise 异常
process.on('unhandledRejection', (reason, promise) => {
  console.error('未处理Promise异常:', promise, 'reason:', reason)
  // 这里可以添加自定义的异常处理逻辑
})

# 最佳实践

# 1. 在应用入口添加监听器

在 main.ts 或 app.js 等应用入口文件中尽早添加异常监听：

// 应用启动前设置
process.on('uncaughtException', handleUncaughtException)
process.on('unhandledRejection', handleUnhandledRejection)

function handleUncaughtException(error) {
  console.error('未捕获的异常:', error)
  // 记录到日志系统
  logger.error('Uncaught Exception', error)
  // 可能需要重启进程
  process.exit(1)
}

function handleUnhandledRejection(reason, promise) {
  console.error('未处理的Promise拒绝:', reason)
  // 记录到日志系统
  logger.error('Unhandled Rejection', { reason, promise })
}

# 2. 结合日志系统

在生产环境中，应该将这些异常记录到专业的日志系统：

const logger = require('./logger') // 你的日志模块

process.on('unhandledRejection', (reason, promise) => {
  logger.error({
    type: 'UnhandledRejection',
    reason: reason,
    stack: reason?.stack,
    promise: promise
  })
})

# 3. 发送告警通知

对于生产环境，建议在捕获异常时发送告警：

process.on('unhandledRejection', async (reason, promise) => {
  console.error('未处理Promise异常:', reason)
  
  // 记录日志
  logger.error('Unhandled Rejection', reason)
  
  // 发送告警（钉钉、邮件、短信等）
  await sendAlert({
    level: 'critical',
    message: '应用出现未处理的Promise异常',
    error: reason,
    stack: reason?.stack
  })
})

# 注意事项

  1. 不要忽略异常：捕获异常后一定要进行处理，不能简单地吞掉错误
  2. 考虑进程重启：对于致命错误，可能需要优雅地重启进程
  3. 使用 PM2 等进程管理器：在生产环境中使用进程管理器可以自动重启崩溃的进程
  4. 完善错误处理：尽量在代码中添加 try-catch 和 .catch()，从源头减少未捕获异常

# 小结

未处理的 Promise 异常是 Node.js 应用中的常见问题。通过添加全局异常监听器，结合日志系统和告警机制，可以更好地发现和定位问题。但更重要的是，在编码时就要注意正确处理 Promise 异常，从根本上避免这类问题的发生。
