# JavaScript中实现Sleep函数的两种方法

在一些场景下，我们需要让程序暂停执行一段时间。虽然 JavaScript 没有内置的 sleep 函数，但可以通过 Promise 和 async/await 轻松实现。

# 使用 Promise 实现

这是最常见的实现方式，使用 Promise 和 setTimeout：

function sleep(ms) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve(1);
    }, ms);
  });
}

// 使用示例
await sleep(1000); // 暂停 1 秒

# 使用 Thenable 对象实现

这是一种更简洁的实现，不使用 Promise，而是返回一个 thenable 对象：

function sleep2(ms) {
  return {
    then(resolve) {
      setTimeout(resolve, ms);
    }
  };
}

// 使用示例
await sleep2(1000); // 暂停 1 秒

# 两种方法的区别

Promise 方式：

  • 返回标准的 Promise 对象
  • 可以使用 .then() 和 .catch() 链式调用
  • 更符合现代 JavaScript 编程习惯
  • 支持 Promise 的所有特性（如 Promise.all、Promise.race 等）

Thenable 方式：

  • 代码更简洁
  • 只实现了 then 方法的最小接口
  • 仍然可以与 await 配合使用
  • 不支持 Promise 的其他特性

# 使用场景

这两种方法在大多数场景下都能正常工作，常见的使用场景包括：

  • 模拟延迟操作
  • 控制请求频率（避免接口被限流）
  • 动画效果的时序控制
  • 轮询操作的间隔等待

# 实际应用示例

// 示例1：延迟执行
async function delayedGreeting() {
  console.log('Hello');
  await sleep(2000);
  console.log('World');
}

// 示例2：控制请求频率
async function batchRequest(urls) {
  for (let url of urls) {
    await fetch(url);
    await sleep(100); // 每个请求间隔 100ms
  }
}

// 示例3：重试机制
async function retryWithDelay(fn, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await sleep(1000 * (i + 1)); // 递增延迟
    }
  }
}

# 注意事项

  • sleep 不会阻塞整个 JavaScript 运行时，只会暂停当前的 async 函数
  • 在浏览器环境中，过度使用 sleep 可能影响用户体验
  • Node.js 环境中，对于 CPU 密集型任务，sleep 无法让出 CPU 时间

# 小结

这两种 sleep 实现方式各有特点，推荐使用 Promise 方式，因为它更加通用和标准。Thenable 方式则更适合追求代码简洁的场景。根据实际需求选择合适的实现即可。
