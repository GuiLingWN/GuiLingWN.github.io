# Node.js版本管理与环境配置

在Node.js开发过程中，经常需要在不同项目间切换Node版本，或者需要配置国内镜像源来加速依赖下载。这里整理了一些实用的环境配置技巧。

# NVM版本管理工具

NVM (Node Version Manager) 是Node.js的版本管理工具，可以方便地在多个Node版本之间切换。

# 安装与配置

如果遇到问题，建议下载最新版本重新安装：

  • NVM for Windows发布页 (opens new window)

# 配置国内镜像源

修改配置文件 C:\Users\Administrator\AppData\Roaming\nvm\settings.txt，添加以下内容：

node_mirror: https://npmmirror.com/mirrors/node/
npm_mirror: https://npmmirror.com/mirrors/npm/

# 常用命令

查看已安装的版本：

nvm list
# 或查看所有可下载的版本
nvm list available

安装指定版本：

nvm install 11.13.0
# 或安装最新稳定版
nvm install latest

切换Node版本：

nvm use 11.13.0

卸载版本：

nvm uninstall 11.13.0

# 其他实用命令

  • nvm arch：显示当前node运行在32位还是64位
  • nvm on：开启node版本管理
  • nvm off：关闭node版本管理
  • nvm proxy [url]：设置下载代理
  • nvm root [path]：设置node版本存储目录
  • nvm version：显示nvm版本

# NPM镜像源配置

# 设置淘宝镜像

NPM默认从国外服务器下载依赖，速度较慢。可以切换到国内镜像：

npm config set registry https://registry.npmmirror.com

# 验证配置

npm config get registry
# 或
npm info express

# 恢复官方源

如需恢复到NPM官方源：

npm config set registry https://registry.npmjs.org

# 重要提醒

淘宝NPM镜像已更新域名：

  • 新域名：https://npmmirror.com
  • 新Registry：https://registry.npmmirror.com
  • 旧域名（已停用）：npm.taobao.org、registry.npm.taobao.org

# PM2进程管理配置

PM2是Node.js应用的进程管理器，常用于生产环境部署。

# Linux系统开机自启

pm2 startup
pm2 save

这会在/etc/init.d/目录生成启动脚本，并将当前运行的应用保存到/root/.pm2/dump.pm2。

# 日志管理

PM2默认日志位置：C:\Users\Administrator\.pm2\logs（Windows）

日志轮转配置（pm2-logrotate-ext）：

pm2 install pm2-logrotate-ext
pm2 set pm2-logrotate-ext:max_size 30M

# 每周轮转一次
pm2 set pm2-logrotate-ext:rotateInterval '0 0 * * 1'

# 每天轮转一次
pm2 set pm2-logrotate-ext:rotateInterval '0 0 * * *'

# JSON配置文件示例

{
  "apps": [{
    "name": "my-app",
    "script": "./app.js",
    "log_date_format": "YYYY-MM-DD HH:mm:ss:SS",
    "log_file": "C:/logs/app.log",
    "error_file": "C:/logs/app_err.log",
    "env": {
      "NODE_ENV": "production"
    },
    "instances": 2,
    "exec_mode": "cluster",
    "watch": false
  }]
}

# 禁用默认日志

如果不想在~/.pm2/logs/目录生成日志：

module.exports = {
  apps: [{
    name: "my-app",
    script: "app.js",
    error: "/logs/my-app-error.log",
    output: "/logs/my-app-out.log",
    disable_logs: true  // 关键配置
  }]
};

# Node-fetch使用注意事项

使用node-fetch发起HTTP请求时，需要注意：

默认情况下，headers会使用node-fetch的默认值：

  • user-agent: 'node-fetch/1.0 (+https://github.com/bitinn/node-fetch)'

如需模拟真实浏览器请求，需要手动覆盖Host和user-agent等headers。

# Create React App加速

使用create-react-app创建项目时，如果下载速度慢，同样可以设置淘宝镜像：

npm config set registry https://registry.npm.taobao.org
# 验证
npm config get registry

# 小结

合理配置Node.js开发环境，可以显著提升开发效率：

  • 使用NVM管理多个Node版本
  • 配置国内镜像源加速依赖下载
  • 使用PM2进行生产环境进程管理和日志管理

这些配置一次设置，长期受益。
