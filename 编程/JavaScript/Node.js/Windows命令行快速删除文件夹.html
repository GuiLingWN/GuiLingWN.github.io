# Windows命令行快速删除文件夹

在 Windows 系统中删除大型文件夹（如 node_modules）时，使用资源管理器往往非常慢。这时候命令行工具就能派上用场，速度快得多。

# 问题场景

开发前端项目时，node_modules 文件夹通常包含成千上万个小文件。使用鼠标右键删除可能需要等待很长时间，甚至可能卡死。

# 解决方案

# CMD 命令

在命令提示符（CMD）中，使用 rd 命令可以快速删除：

rd /s /q node_modules

参数说明：

  • rd - remove directory 的缩写，删除目录命令
  • /s - 删除目录及其所有子目录和文件
  • /q - 安静模式，不询问确认

# PowerShell 命令

如果你使用的是 PowerShell，命令略有不同：

rd -r node_modules

参数说明：

  • rd - 删除目录命令（Remove-Item 的别名）
  • -r - 递归删除（等同于 -Recurse）

# 使用建议

  1. 谨慎使用：这些命令不会移到回收站，删除后无法恢复
  2. 确认路径：使用前先用 cd 命令确认当前目录
  3. 创建快捷方式：可以创建 .bat 脚本或 PowerShell 脚本快速执行
  4. 避免误删：建议先用 dir 或 ls 命令查看目录内容

# 扩展技巧

# 创建批处理脚本

创建 clean-node-modules.bat 文件：

@echo off
echo 正在删除 node_modules 文件夹...
rd /s /q node_modules
echo 删除完成！
pause

# PowerShell 脚本

创建 clean-node-modules.ps1 文件：

Write-Host "正在删除 node_modules 文件夹..." -ForegroundColor Yellow
Remove-Item -Recurse -Force node_modules
Write-Host "删除完成！" -ForegroundColor Green

# 性能对比

在我的测试中，对于一个约 500MB、包含 50000+ 文件的 node_modules 文件夹：

  • 资源管理器删除：约 2-5 分钟
  • 命令行删除：约 10-30 秒

命令行方式的速度提升非常明显。

# 小结

掌握这两个简单的命令，可以大幅提升删除大型文件夹的效率。特别是在频繁切换项目或清理磁盘空间时，能节省不少时间。
