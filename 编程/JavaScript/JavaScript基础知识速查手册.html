# JavaScript基础知识速查手册

本文整理了JavaScript开发中常用的基础知识点和技巧，包括数组操作、字符串处理、URL解析等，方便日常开发查阅。

# 数组操作

# 数组方法：队列和栈的实现

JavaScript数组提供了多个方法来实现队列和栈的操作：

# 基本方法

  • shift()：从数组开头删除第一个元素，并返回该元素
  • unshift()：在数组开头添加一个或多个元素，并返回新数组长度
  • push()：在数组末尾添加元素，并返回新数组长度
  • pop()：从数组末尾删除最后一个元素，并返回该元素
  • reverse()：颠倒数组中元素的顺序

# 性能注意事项

  1. unshift()比push()慢差不多100倍，因为需要移动所有元素
  2. reverse()方法性能极高，可以放心使用
  3. 如果需要频繁在开头添加元素，可以先push()再reverse()
// 队列（先进先出）
let queue = [];
queue.push(1);      // 入队
queue.push(2);
let first = queue.shift();  // 出队

// 栈（后进先出）
let stack = [];
stack.push(1);      // 入栈
stack.push(2);
let last = stack.pop();     // 出栈

// 高效的unshift替代方案
let arr = [1, 2, 3];
arr.push(0);        // [1, 2, 3, 0]
arr.reverse();      // [0, 3, 2, 1] - 实现了类似unshift(0)的效果

# 数组迭代：every和some

every()和some()是两个常用的数组迭代方法：

  • every()：对数组每一项运行函数，所有项返回true时才返回true
  • some()：对数组每一项运行函数，任一项返回true就返回true
let arr = [1, 2, 3, 4, 5];

// every：判断是否所有元素都大于0
let allPositive = arr.every(item => item > 0);  // true

// some：判断是否存在偶数
let hasEven = arr.some(item => item % 2 === 0);  // true

# 性能优化

两个方法都有短路特性：

  • some()遇到第一个返回true的值就停止遍历
  • every()遇到第一个返回false的值就停止遍历

# 高阶函数：map和reduce

# map()

将数组的每个元素按照规则转换：

let arr = [1, 2, 3, 4, 5];
let squared = arr.map(x => x * x);  // [1, 4, 9, 16, 25]

# reduce()

把函数作用在数组上进行累积计算：

let arr = [1, 2, 3, 4, 5];
let sum = arr.reduce((prev, curr) => prev + curr, 0);  // 15

// 数组求和
let total = arr.reduce((acc, val) => acc + val, 0);

// 数组转对象
let arr2 = ['a', 'b', 'c'];
let obj = arr2.reduce((acc, val, index) => {
    acc[val] = index;
    return acc;
}, {});  // {a: 0, b: 1, c: 2}

# 字符串操作

# 字符串转字符数组

function formStr(str) {
    var strArr = [];
    for (var i = 0; i < str.length; i++) {
        strArr.push(str[i]);
    }
    return strArr;
}

// 示例
formStr("hello世界");  // ["h", "e", "l", "l", "o", "世", "界"]

// ES6简化版
const toCharArray = str => [...str];
// 或者
const toCharArray2 = str => Array.from(str);

# URL参数解析

从URL中提取查询参数：

function GetRequest() {
    var url = location.search;  // 获取url中"?"符后的字符串
    var theRequest = new Object();
    if (url.indexOf("?") != -1) {
        var str = url.substr(1);
        strs = str.split("&");
        for(var i = 0; i < strs.length; i++) {
            console.log(strs[i]);
            theRequest[strs[i].split("=")[0]] = unescape(strs[i].split("=")[1]);
        }
    }
    return theRequest;
}

// 使用示例
var Request = GetRequest();
var username = Request['username'] || 'default';
var password = Request['password'] || 'default';

# 现代化写法

// 使用URLSearchParams API（推荐）
const params = new URLSearchParams(location.search);
const username = params.get('username') || 'default';
const password = params.get('password') || 'default';

// 获取所有参数
const allParams = Object.fromEntries(params);

# 变量交换技巧

三种交换两个变量值的方法：

# 方法一：传统方法（最易理解）

var a = 1, b = 2;
a = a + b;  // a = 3
b = a - b;  // b = 1
a = a - b;  // a = 2
console.log(a, b);  // 2 1

# 方法二：一行实现

var a = 1, b = 2;
a = b + a - (b = a);
console.log(a, b);  // 2 1

# 方法三：位运算（异或）

var a = 1, b = 2;
a = a ^ b;
b = a ^ b;
a = a ^ b;
console.log(a, b);  // 2 1

# 方法四：ES6解构赋值（最推荐）

let a = 1, b = 2;
[a, b] = [b, a];
console.log(a, b);  // 2 1

# JavaScript模块化规范

# CommonJS

  • 特点：同步加载，主要用于服务端（Node.js）
  • 加载方式：同步
  • 使用场景：服务器端，模块文件在本地硬盘
// 导出
module.exports = { foo, bar };

// 导入
const { foo, bar } = require('./module');

# AMD（Asynchronous Module Definition）

  • 特点：异步加载，依赖前置
  • 使用场景：浏览器环境
  • 代表：RequireJS
// 定义模块
define(['dep1', 'dep2'], function(dep1, dep2) {
    return { foo, bar };
});

// 使用模块
require(['module'], function(module) {
    module.foo();
});

# CMD（Common Module Definition）

  • 特点：异步加载，就近依赖
  • 使用场景：浏览器环境
  • 代表：Sea.js
// 定义模块
define(function(require, exports, module) {
    var dep = require('./dep');
    exports.foo = function() {};
});

# ES6 Module

  • 特点：编译时加载，输出值的引用
  • 使用场景：浏览器和服务器通用
  • 优势：语言标准层面的支持
// 导出
export const foo = 'bar';
export default function() {}

// 导入
import { foo } from './module';
import defaultExport from './module';

# 主要区别

规范 加载方式 使用环境 特点
CommonJS 同步 服务端 值的拷贝
AMD 异步 浏览器 依赖前置
CMD 异步 浏览器 就近依赖
ES6 Module 编译时 通用 值的引用

# 堆和栈的区别

# 基本数据类型（存放在栈中）

  • 存放在栈内存中的简单数据段
  • 数据大小确定，内存空间大小可以分配
  • 直接按值存放，可以直接访问

# 引用类型（存放在堆中）

  • 栈（stack）：自动分配的内存空间，由系统自动释放
  • 堆（heap）：动态分配的内存，大小不定也不会自动释放
// 基本类型（栈）
let a = 10;
let b = a;
b = 20;
console.log(a);  // 10 - 互不影响

// 引用类型（堆）
let obj1 = { value: 10 };
let obj2 = obj1;
obj2.value = 20;
console.log(obj1.value);  // 20 - 共享引用

# 小结

掌握这些JavaScript基础知识可以帮助我们写出更高效、更优雅的代码。数组和字符串操作是日常开发的基础，模块化规范的理解有助于组织代码结构，堆栈概念则是理解JavaScript内存管理的关键。在实际开发中，建议使用ES6+的新特性来简化代码，提高可读性。
