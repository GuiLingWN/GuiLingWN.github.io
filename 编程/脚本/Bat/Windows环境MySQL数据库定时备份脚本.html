# Windows环境MySQL数据库定时备份脚本

数据库定期备份是一项重要但容易被忽视的工作。在 Windows 环境下，一个简单的 BAT 脚本配合任务计划程序，就能实现自动化备份。

# 备份脚本

以下是一个自动备份 MySQL 数据库的批处理脚本：

@echo off
set dbs=(bcy,nlwb)
set today=%date:~,4%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%
for %%i in %dbs% do F:/wn/env/mysql-5.6.21-winx64/bin/mysqldump --opt -u root --password= %%i > F:/wn/backup/mysql/%%i_%today%.sql
@echo on

# 脚本说明

变量配置：

  • dbs - 需要备份的数据库名称列表，用逗号分隔
  • today - 生成带时间戳的文件名，格式为 YYYYMMDD_HHMM

备份命令：

  • 使用 mysqldump 工具导出数据库
  • --opt 参数启用优化选项（包括锁表、快速导出等）
  • -u root 指定数据库用户
  • --password= 后面可以填写密码（为安全起见，建议留空或使用其他方式管理密码）
  • 输出文件名包含数据库名和时间戳

# 使用方法

  1. 修改脚本参数

    • 将 dbs 列表改为你需要备份的数据库
    • 修改 MySQL 安装路径为你的实际路径
    • 修改备份输出目录
    • 根据需要设置数据库密码

  2. 保存脚本

    • 将脚本保存为 .bat 文件，如 mysql_backup.bat

  3. 设置定时任务

    • 打开"任务计划程序"（Task Scheduler）
    • 创建新任务，设置触发器（如每天凌晨执行）
    • 选择执行的脚本文件

# 安全建议

  • 密码安全：不建议在脚本中明文存储密码，可以考虑使用 MySQL 配置文件存储凭据
  • 备份验证：定期检查备份文件是否正常，建议测试恢复流程
  • 备份保留：建议添加清理旧备份的逻辑，避免磁盘空间耗尽
  • 异地备份：关键数据建议同步到其他服务器或云存储

# 扩展功能

可以在脚本基础上添加以下功能：

  • 备份文件压缩（使用 7zip 等工具）
  • 自动清理 N 天前的旧备份
  • 备份完成后发送邮件通知
  • 上传到云存储服务

# 小结

这个简单的脚本可以帮你实现 Windows 环境下的 MySQL 数据库自动备份。虽然功能基础，但足以应对大多数中小型项目的备份需求。记得定期检查备份文件的有效性，并根据实际情况调整备份策略。
