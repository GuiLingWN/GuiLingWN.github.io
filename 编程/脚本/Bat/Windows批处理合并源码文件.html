# Windows批处理合并源码文件

在某些场景下（如软件著作权申请），需要将项目中的所有源代码合并成一个文件。手动复制粘贴既费时又容易出错，这时候一个简单的批处理脚本就能帮我们自动完成。

# 使用场景

  • 软件著作权申请需要提交源代码文件
  • 代码审查或分享时需要合并多个文件
  • 备份或归档项目代码
  • 生成代码文档

# 批处理脚本

以下是一个通用的源码合并脚本：

@echo off
REM 文件过滤器
set file_filter=*.ts *.js *.json *.map *.css

REM 指定原目录
set src_dir=D:\\workspace\\软著代码合并\\file\\web
REM 指定输出文件
set output_file=D:\\workspace\\软著代码合并\\web.log

cd %src_dir%
del %output_file%
for /F %%i in ('dir /b/s/a:-d %file_filter%') do (
    type %%i>>%output_file%
)

# 脚本说明

# 配置项

1. 文件过滤器

set file_filter=*.ts *.js *.json *.map *.css

指定要合并的文件类型。可以根据需要添加或删除文件扩展名。

2. 源目录

set src_dir=D:\\workspace\\软著代码合并\\file\\web

指定要扫描的源代码目录。注意路径中的 \ 需要使用 \\ 转义。

3. 输出文件

set output_file=D:\\workspace\\软著代码合并\\web.log

指定合并后的输出文件路径。

# 核心命令

for /F %%i in ('dir /b/s/a:-d %file_filter%') do (
    type %%i>>%output_file%
)

命令解析：

  • dir /b/s/a:-d - 递归查找文件
    • /b - 仅显示文件名
    • /s - 搜索子目录
    • /a:-d - 排除目录，只显示文件
  • type %%i>>%output_file% - 将文件内容追加到输出文件

# 使用方法

  1. 创建脚本文件

    • 将代码保存为 merge-source.bat

  2. 修改配置

    • 修改 file_filter 为需要的文件类型
    • 修改 src_dir 为项目源代码目录
    • 修改 output_file 为输出文件路径

  3. 执行脚本

    • 双击运行 merge-source.bat
    • 等待执行完成，查看输出文件

# 改进版本

# 添加文件分隔符

为了在合并文件中区分不同的源文件，可以添加分隔符：

@echo off
set file_filter=*.ts *.js *.json
set src_dir=D:\\workspace\\project\\src
set output_file=D:\\workspace\\merged.txt

cd %src_dir%
if exist %output_file% del %output_file%

for /F %%i in ('dir /b/s/a:-d %file_filter%') do (
    echo. >> %output_file%
    echo ==================== >> %output_file%
    echo 文件: %%i >> %output_file%
    echo ==================== >> %output_file%
    type %%i >> %output_file%
    echo. >> %output_file%
)

echo 合并完成！共处理文件：
dir /b/s/a:-d %file_filter% | find /c /v ""

# 添加进度显示

@echo off
set file_filter=*.ts *.js *.json
set src_dir=D:\\workspace\\project\\src
set output_file=D:\\workspace\\merged.txt

cd %src_dir%
if exist %output_file% del %output_file%

echo 开始合并文件...
set count=0

for /F %%i in ('dir /b/s/a:-d %file_filter%') do (
    set /a count+=1
    echo 正在处理第 !count! 个文件: %%~nxi
    type %%i >> %output_file%
)

echo.
echo 合并完成！共处理 %count% 个文件

# 注意事项

  1. 字符编码：合并后的文件编码可能不一致，建议使用支持多种编码的编辑器打开
  2. 文件大小：如果项目很大，合并后的文件可能非常大，注意磁盘空间
  3. 路径问题：Windows 路径中的反斜杠需要转义为 \\
  4. 权限问题：确保对源目录和输出目录有读写权限

# 小结

这个批处理脚本虽然简单，但在需要合并源代码文件时非常实用。特别是在申请软件著作权时，能够快速生成符合要求的代码文件。根据实际需求修改文件过滤器和输出格式，就能适应各种场景。
