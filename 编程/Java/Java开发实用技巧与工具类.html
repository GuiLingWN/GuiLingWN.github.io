# Java开发实用技巧与工具类

在Java日常开发中，经常需要处理字节数组、数值转换等基础操作。本文整理了一些实用的工具方法和开发中遇到的问题解决方案。

# 字节数组操作

# 合并两个byte数组

在处理网络通信或文件操作时，经常需要合并多个字节数组：

/**
 * 合并两个byte数组
 * @param byte_1 第一个数组
 * @param byte_2 第二个数组
 * @return 合并后的数组
 */
private static byte[] byteMerger(byte[] byte_1, byte[] byte_2) {
    byte[] byte_3 = new byte[byte_1.length + byte_2.length];
    System.arraycopy(byte_1, 0, byte_3, 0, byte_1.length);
    System.arraycopy(byte_2, 0, byte_3, byte_1.length, byte_2.length);
    return byte_3;
}

# int与byte[]之间的转换

# 方式一：低位在前，高位在后

/**  
 * 将int数值转换为占四个字节的byte数组，本方法适用于(低位在前，高位在后)的顺序
 * @param value 要转换的int值 
 * @return byte数组 
 */    
public static byte[] intToBytes(int value) {   
    byte[] src = new byte[4];  
    src[3] = (byte) ((value>>24) & 0xFF);  
    src[2] = (byte) ((value>>16) & 0xFF);  
    src[1] = (byte) ((value>>8) & 0xFF);    
    src[0] = (byte) (value & 0xFF);                  
    return src;   
}

# 方式二：高位在前，低位在后

/**  
 * 将int数值转换为占四个字节的byte数组，本方法适用于(高位在前，低位在后)的顺序
 * @param value 要转换的int值
 * @return byte数组
 */    
public static byte[] intToBytes2(int value) {   
    byte[] src = new byte[4];  
    src[0] = (byte) ((value>>24) & 0xFF);  
    src[1] = (byte) ((value>>16) & 0xFF);  
    src[2] = (byte) ((value>>8) & 0xFF);    
    src[3] = (byte) (value & 0xFF);       
    return src;  
}

这两种方式的区别在于字节序（大端序和小端序），选择时需要根据实际的通信协议或文件格式要求来决定。

# WebSocket连接问题

在使用WebSocket时，如果客户端始终连接不上服务器，一直报404错误，很可能是连接地址配置错误。

# 正确的WebSocket地址格式

本地测试的WebSocket地址应该遵循以下格式：

ws://localhost:8080/项目名/监听地址

其中：

  • 项目名：不是WebSocket的类名，而是项目的context路径
  • 监听地址：是@ServerEndpoint注解中定义的值

# 示例

// 正确的连接方式
new WebSocket("ws://localhost:8080/websocketTest/websocket");

这个看似简单的地址配置问题，在实际开发中很容易被忽略，导致浪费大量调试时间。记住地址的构成规则，就能快速定位和解决此类问题。

# 小结

这些工具方法和技巧虽然简单，但在实际开发中非常实用。字节数组操作在网络编程和文件处理中经常用到，WebSocket地址配置则是建立实时通信的基础。将这些常用操作封装成工具类，可以提高开发效率，减少重复代码。
