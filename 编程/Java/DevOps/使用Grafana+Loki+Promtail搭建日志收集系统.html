# 使用Grafana+Loki+Promtail搭建日志收集系统

在微服务架构中，集中化的日志管理非常重要。Grafana+Loki+Promtail是一套轻量级的日志收集和查询方案。本文记录了使用Docker部署这套系统的完整过程。

# 系统架构

  • Loki：日志聚合系统，类似Prometheus但专注于日志
  • Promtail：日志采集器，负责收集和推送日志到Loki
  • Grafana：可视化界面，用于查询和展示日志

# Docker部署方式

# 1. 安装Loki日志采集插件（可选）

如果需要采集Docker容器日志，可以安装Loki的Docker插件：

# 安装插件
docker plugin install \
  docker.mirrors.sjtug.sjtu.edu.cn/grafana/loki-docker-driver:latest \
  --alias loki \
  --grant-all-permissions

# 2. 配置Docker全局日志驱动（可选）

编辑Docker daemon配置文件：

# 编辑配置文件
vi /etc/docker/daemon.json

添加以下配置：

{
    "log-driver": "loki",
    "log-opts": {
        "loki-url": "http://192.168.2.9:3100/loki/api/v1/push",
        "max-size": "50m",
        "max-file": "10"
    }
}

注意：修改后需要重启Docker服务。

# 3. 部署Loki

# 创建数据目录
mkdir -p /data/docker/loki
mkdir -p /data/docker/loki/index
mkdir -p /data/docker/loki/chunks

# 设置目录权限
chmod -R 777 /data/docker/loki/index
chmod -R 777 /data/docker/loki/chunks

# 运行Loki容器
docker run -d \
  --restart=always \
  --name loki \
  --privileged=true \
  -v /data/docker/loki:/opt/loki \
  -p 3100:3100 \
  grafana/loki:2.8.0 -config.file=/opt/loki/loki-config.yaml

# 4. 部署Promtail

# 创建配置目录
mkdir -p /data/docker/promtail

# 运行Promtail容器
docker run -d \
  --restart=always \
  --name promtail \
  --privileged=true \
  -v /data/docker/promtail:/mnt/config \
  -v /data/logs:/var/log \
  -v /var/log/nginx:/var/log/nginx \
  grafana/promtail -config.file=/mnt/config/promtail-config.yaml

# 5. 部署Grafana

docker run -d \
  --restart=always \
  --name grafana \
  -p 3000:3000 \
  grafana/grafana-enterprise:9.2.15

# 6. 部署Prometheus（可选）

如果需要配合Prometheus使用：

mkdir -p /data/docker/prometheus

docker run -d \
  -p 9090:9090 \
  -v /data/docker/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus

# 配置文件

# Loki配置文件

需要在/data/docker/loki/loki-config.yaml中添加配置。如果遇到权限问题，需要添加wal目录配置：

ingester:
  wal:
    enabled: true
    dir: /loki/wal

# Promtail配置文件

配置文件位置：/data/docker/promtail/promtail-config.yaml

可以参考官方配置示例进行定制。

# 常见问题处理

# 权限错误

如果遇到以下错误：

msg="error running loki" err="mkdir wal: permission denied"

解决方法：

  1. 在配置文件中增加wal目录配置
  2. 确保目录权限正确（777或容器内用户可写）

# 日志管理

# 手动清理历史日志（loki自带的日志清理配置功能经常不生效，所以手动清理）

随着时间推移，日志文件会占用大量磁盘空间，需要定期清理：

# 统计目录文件数量
find /data/docker/loki/chunks/ -type f | wc -l

# 查询最早文件创建时间
find /data/docker/loki/chunks/ -type f -printf '%T@ %p\n' | \
  sort -n | head -1 | \
  awk '{print strftime("%Y-%m-%d %H:%M:%S", $1)}'

# 列出一年前的文件
find /data/docker/loki/chunks/ -type f -mtime +365

# 删除一年前的文件
find /data/docker/loki/chunks/ -type f -mtime +365 -exec rm {} \;

# 统计100天前文件的总大小（GB）
find /data/docker/loki/chunks/ -type f -mtime +100 -exec du -b {} + | \
  awk '{ total += $1 } END { print total/1024/1024/1024 }'

# 统计100天前的文件总数
find /data/docker/loki/chunks/ -type f -mtime +100 | wc -l

# 移动100天前的文件到备份目录
find /data/docker/loki/chunks/ -type f -mtime +100 \
  -exec mv {} /data/docker/loki/backup/ \;

# 按月份统计文件

find /data/docker/loki/chunks/ -type f -printf '%TY-%Tm %s\n' | awk '
{
    size[$1] += $2
    count[$1]++
}
END {
    printf "%-10s %-10s %-15s %-15s\n", "Month", "Count", "Total Size", "Avg Size"
    printf "%-10s %-10s %-15s %-15s\n", "----------", "----------", "---------------", "---------------"
    for (month in size) {
        total_gb = size[month] / 1024 / 1024 / 1024
        avg_kb = count[month] > 0 ? (size[month] / count[month] / 1024) : 0
        printf "%-10s %-10d %-10.2f GB %-10.2f KB\n", month, count[month], total_gb, avg_kb
    }
}' | sort

# 小结

Grafana+Loki+Promtail提供了一套轻量级且功能强大的日志解决方案，相比ELK栈占用资源更少。通过Docker部署简化了安装过程，但需要注意配置文件和权限问题。定期清理历史日志可以有效控制磁盘占用，保持系统稳定运行。
