# Jenkins容器化部署与SSH乱码解决

Jenkins是一款流行的持续集成工具，使用Docker部署Jenkins可以简化环境配置。本文记录了Jenkins容器化部署的步骤和SSH控制台乱码问题的解决方案。

# Docker部署Jenkins

# 基本部署步骤

# 1. 创建Jenkins工作目录
mkdir /data/jenkins_home/

# 2. 修改目录所有者，使Jenkins容器能够操作该目录
#    UID 1000是Jenkins容器内的默认用户ID
chown -R 1000:1000 /data/jenkins_home/

# 3. 运行Jenkins容器
docker run -d \
  --name jenkins \
  -p 8080:8080 \
  -p 50000:50000 \
  -v /data/jenkins_home:/var/jenkins_home \
  jenkins/jenkins:jdk17

# 参数说明

  • -d：后台运行容器
  • --name jenkins：指定容器名称
  • -p 8080:8080：映射Web访问端口
  • -p 50000:50000：映射Agent通信端口
  • -v：挂载数据卷，持久化Jenkins数据
  • jenkins/jenkins:jdk17：使用JDK17版本的Jenkins镜像

# 配置时区的部署方式

如果需要设置容器时区为亚洲/上海时区：

mkdir /data/docker/jenkins_home/
chown -R 1000:1000 /data/docker/jenkins_home/

docker run -d \
  --name jenkins17 \
  -p 8081:8080 \
  -e TZ=Asia/Shanghai \
  -v /data/docker/jenkins_home:/var/jenkins_home \
  jenkins/jenkins:jdk17

# 部署注意事项

  1. 目录权限：一定要设置正确的目录所有者（UID 1000），否则Jenkins无法写入数据
  2. 端口冲突：确保8080和50000端口未被占用
  3. 数据持久化：务必挂载数据卷，避免容器重启后数据丢失
  4. 初始密码：首次访问需要查看初始管理员密码
# 查看初始密码
cat /data/jenkins_home/secrets/initialAdminPassword

# Jenkins SSH控制台乱码问题

在Jenkins执行SSH远程命令时，如果输出包含中文，可能会在控制台看到乱码。这是由于字符编码不一致导致的。

# 问题现象

执行包含中文输出的脚本时，Jenkins控制台显示乱码字符。

# 解决方案

在Jenkins的SSH执行命令中，通过管道使用iconv命令转换字符编码：

bash ./publish.sh | iconv -f utf-8 -t gbk

# 参数说明

  • -f utf-8：指定源编码为UTF-8
  • -t gbk：转换目标编码为GBK
  • |：管道符，将前一个命令的输出传递给后一个命令

# 其他编码转换场景

根据实际情况，可能需要不同的编码转换：

# UTF-8转GBK
command | iconv -f utf-8 -t gbk

# GBK转UTF-8
command | iconv -f gbk -t utf-8

# 查看系统支持的编码
iconv --list

# 小结

使用Docker部署Jenkins简化了环境配置，但需要注意目录权限和端口映射等细节。SSH控制台乱码问题虽然不影响功能，但会降低日志的可读性，通过简单的编码转换就能解决。在实际使用中，还需要根据项目需求配置构建环境、安装插件等，这些配置都会保存在挂载的数据卷中，方便迁移和备份。
