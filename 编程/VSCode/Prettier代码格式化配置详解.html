# Prettier代码格式化配置详解

Prettier是一个流行的代码格式化工具，支持多种编程语言。通过配置文件可以自定义格式化规则，保持团队代码风格的一致性。

# 配置文件

在项目根目录创建.prettierrc或.prettierrc.json文件，添加以下配置：

{
  "printWidth": 120,
  "semi": true,
  "singleQuote": true,
  "trailingComma": "none",
  "bracketSpacing": true,
  "jsxBracketSameLine": false,
  "arrowParens": "avoid",
  "requirePragma": false,
  "proseWrap": "preserve"
}

# 配置项说明

# printWidth

  • 默认值: 80
  • 说明: 一行代码的最大字符数，超过此长度会自动换行
  • 建议: 120或100，根据团队显示器和个人喜好调整
// printWidth: 80
const example = veryLongFunctionName(
  argument1,
  argument2,
  argument3
);

// printWidth: 120
const example = veryLongFunctionName(argument1, argument2, argument3);

# semi

  • 默认值: true
  • 说明: 是否在语句末尾添加分号
  • 选项: true | false
// semi: true
const a = 1;
const b = 2;

// semi: false
const a = 1
const b = 2

# singleQuote

  • 默认值: false
  • 说明: 是否使用单引号代替双引号
  • 选项: true | false
// singleQuote: true
const str = 'hello';

// singleQuote: false
const str = "hello";

# trailingComma

  • 默认值: "es5"
  • 说明: 对象或数组最后一个元素后是否添加逗号
  • 选项: "none" | "es5" | "all"
// trailingComma: "none"
const obj = {
  a: 1,
  b: 2
};

// trailingComma: "es5"
const obj = {
  a: 1,
  b: 2,
};

// trailingComma: "all"
const fn = (
  arg1,
  arg2,
) => {};

# bracketSpacing

  • 默认值: true
  • 说明: 对象字面量的括号之间是否添加空格
  • 选项: true | false
// bracketSpacing: true
const obj = { a: 1, b: 2 };

// bracketSpacing: false
const obj = {a: 1, b: 2};

# jsxBracketSameLine

  • 默认值: false
  • 说明: JSX标签的闭合括号>是否与最后一行属性放在同一行
  • 选项: true | false
// jsxBracketSameLine: false
<Button
  color="primary"
  size="large"
>
  Click Me
</Button>

// jsxBracketSameLine: true
<Button
  color="primary"
  size="large">
  Click Me
</Button>

# arrowParens

  • 默认值: "always"
  • 说明: 箭头函数参数是否总是添加括号
  • 选项: "always" | "avoid"
// arrowParens: "avoid"
const fn = x => x * 2;

// arrowParens: "always"
const fn = (x) => x * 2;

# requirePragma

  • 默认值: false
  • 说明: 是否只格式化包含特定注释的文件
  • 选项: true | false

当设置为true时，只有包含以下注释的文件才会被格式化：

/**
 * @prettier
 */

// 或
/**
 * @format
 */

# proseWrap

  • 默认值: "preserve"

  • 说明: Markdown文本换行方式

  • 选项: "always" | "never" | "preserve"

  • always: 超过printWidth就换行

  • never: 不换行

  • preserve: 按原样保留

# VS Code集成

# 安装Prettier扩展

  1. 在VS Code中搜索"Prettier - Code formatter"
  2. 点击安装

# 设置为默认格式化工具

在VS Code的settings.json中添加：

{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}

# 忽略文件

创建.prettierignore文件，指定不需要格式化的文件：

# 依赖
node_modules

# 构建产物
dist
build
*.min.js

# 其他
*.md
.next
.nuxt

# 团队协作建议

  1. 统一配置：将配置文件提交到Git仓库
  2. 编辑器插件：团队成员都安装Prettier插件
  3. Git钩子：使用husky + lint-staged在提交前自动格式化
  4. CI检查：在CI流程中检查代码格式

# 配置pre-commit钩子

// package.json
{
  "husky": {
    "hooks": {
      "pre-commit": "lint-staged"
    }
  },
  "lint-staged": {
    "*.{js,jsx,ts,tsx,json,css,md}": [
      "prettier --write",
      "git add"
    ]
  }
}

# 小结

Prettier通过统一的代码格式化规则，帮助团队保持代码风格一致性，减少代码审查中关于格式的讨论。合理配置Prettier可以提高开发效率，让开发者专注于业务逻辑而非代码格式。建议根据团队习惯和项目特点调整配置，并通过自动化工具确保配置被正确执行。
