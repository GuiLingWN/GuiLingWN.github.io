# VSCode快捷键配置与使用技巧

VSCode是目前最流行的代码编辑器之一，合理配置快捷键可以显著提升编码效率。这里分享一套实用的快捷键配置方案。

# 自定义快捷键配置

在VSCode中，可以通过修改keybindings.json文件来自定义快捷键。以下是一套常用的快捷键配置：

# 注释相关

行注释：

{
  "key": "ctrl+shift+c",
  "command": "editor.action.commentLine",
  "when": "editorTextFocus"
}

块注释：

{
  "key": "ctrl+shift+/",
  "command": "editor.action.blockComment",
  "when": "editorTextFocus"
}

# 编辑操作

删除整行：

{
  "key": "ctrl+d",
  "command": "editor.action.deleteLines",
  "when": "editorTextFocus"
}

触发代码提示：

{
  "key": "alt+/",
  "command": "editor.action.triggerSuggest",
  "when": "editorHasCompletionItemProvider && editorTextFocus && !editorReadonly"
}

# 大小写转换

转大写：

{
  "key": "ctrl+shift+u",
  "command": "editor.action.transformToUppercase"
}

转小写：

{
  "key": "ctrl+shift+d",
  "command": "editor.action.transformToLowercase"
}

注意：这两个快捷键会覆盖原有的功能，需要同时禁用原快捷键：

{
  "key": "ctrl+shift+u",
  "command": "-workbench.action.output.toggleOutput"
},
{
  "key": "ctrl+shift+d",
  "command": "-workbench.view.debug"
}

# 文件操作

保存所有文件：

{
  "key": "ctrl+shift+s",
  "command": "workbench.action.files.saveAll"
}

# 配置方法

  1. 打开VSCode
  2. 按 Ctrl+Shift+P 打开命令面板
  3. 输入 Open Keyboard Shortcuts (JSON)
  4. 将以上配置复制到打开的文件中
  5. 保存文件即可生效

# 快捷键设计思路

这套快捷键配置的设计考虑了以下几点：

习惯延续性：

  • 保留了一些Eclipse、IDEA等IDE的常用快捷键习惯
  • 降低从其他编辑器迁移的学习成本

高频操作优化：

  • 将最常用的操作配置为最容易按的快捷键
  • 如 Ctrl+D 删除行，Alt+/ 代码提示

避免冲突：

  • 在配置新快捷键时，注意禁用原有的冲突快捷键
  • 使用 - 前缀可以禁用原有快捷键

# Prettier格式化配置

VSCode搭配Prettier可以实现代码自动格式化。关于Prettier的详细配置，可以参考项目中的配置文件。

主要配置项包括：

  • 缩进宽度
  • 是否使用分号
  • 引号风格（单引号/双引号）
  • 行尾逗号
  • 行宽限制

# 小结

合理的快捷键配置能够：

  • 减少鼠标操作，提高编码速度
  • 保持专注，不被频繁的菜单操作打断
  • 统一团队成员的操作习惯

建议根据个人习惯和团队规范，调整出最适合自己的快捷键方案。记住常用快捷键需要时间，但一旦养成肌肉记忆，编码效率会有质的提升。
